# Yocto Project 메이저 및 마이너 릴리스 주기 (Release Cadence)

Yocto Project는 예측 가능한 릴리스 일정을 유지하여 개발자와 조직이 프로젝트 계획을 수립할 수 있도록 지원합니다. 릴리스는 크게 메이저 릴리스와 마이너(포인트) 릴리스로 구분됩니다.

## 1. 메이저 릴리스 (Major Release)

Yocto Project는 **6개월** 주기로 새로운 메이저 버전을 릴리스합니다.

*   **일정:** 매년 **4월**과 **10월**에 배포됩니다.
*   **특징:**
    *   새로운 기능 추가, 아키텍처 변경, 최신 커널 및 툴체인(GCC, Glibc 등) 업그레이드가 포함됩니다.
    *   각 릴리스는 고유한 코드명(예: *Kirkstone*, *Scarthgap*, *Nanbield*)을 가집니다.
    *   버전 번호의 첫 번째 또는 두 번째 자리가 변경됩니다 (예: 4.0, 5.0, 5.1).

## 2. 마이너 릴리스 (Minor Release)

각 메이저 릴리스가 배포된 후, 해당 브랜치(Stable Branch)의 안정성을 유지하기 위해 마이너 릴리스(포인트 릴리스)가 제공됩니다.

*   **목적:** 주로 버그 수정, 보안 취약점(CVE) 패치, 그리고 회귀(Regression) 방지에 중점을 둡니다.
*   **특징:**
    *   새로운 기능은 원칙적으로 추가되지 않으며, 기존 기능의 안정화에 주력합니다.
    *   버전 번호의 세 번째 자리가 변경됩니다 (예: 5.0.1, 5.0.2).
    *   안정적인(Stable) 상태로 간주되는 브랜치에서만 발생합니다.

## 3. LTS (Long Term Support)

일부 메이저 릴리스는 **LTS**로 지정되어, 일반적인 릴리스(약 7개월 지원)보다 훨씬 긴 기간(최소 4년) 동안 마이너 릴리스 업데이트를 받습니다. 이는 제품 수명 주기가 긴 임베디드 시스템 개발에 적합합니다.
