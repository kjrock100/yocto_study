# Yocto Project 안정적인 릴리스 프로세스 (Stable Release Process)

Yocto Project의 안정적인 릴리스 프로세스는 메이저 릴리스 이후에도 사용자가 예측 가능하고 안정적인 개발 환경을 유지할 수 있도록 지원하는 체계입니다.

## 1. 안정적인 브랜치 (Stable Branch)

메이저 릴리스(예: 4.0 Kirkstone)가 발표되면, 해당 버전의 소스 코드는 별도의 **Stable Branch**로 관리됩니다. 이 브랜치는 향후 마이너 릴리스(포인트 릴리스)의 기반이 됩니다.

## 2. 백포팅 정책 (Backporting Policy)

안정적인 브랜치에는 마스터 브랜치(개발 중인 최신 버전)에서 검증된 변경 사항만이 선별적으로 적용(Backport)됩니다. 주요 기준은 다음과 같습니다.

*   **포함 대상:**
    *   심각한 버그 수정
    *   보안 취약점(CVE) 패치
    *   빌드 실패 수정
*   **제외 대상:**
    *   새로운 기능 추가
    *   기존 동작을 변경하는 침습적인 수정
    *   API/ABI 호환성을 깨뜨리는 변경

## 3. 릴리스 주기 및 버전링

*   **포인트 릴리스:** 안정적인 브랜치에 수정 사항이 충분히 쌓이거나 긴급한 보안 패치가 필요할 때 마이너 릴리스(예: 4.0.1, 4.0.2)가 배포됩니다.
*   **LTS (Long Term Support):** 특정 메이저 릴리스는 LTS로 지정되어 최소 4년 동안 안정적인 릴리스 프로세스를 통해 지원을 받습니다.

## 4. 품질 보증 (QA)

모든 안정적인 릴리스 후보는 배포 전에 엄격한 테스트 과정을 거칩니다.
*   **AutoBuilder:** Yocto Project의 자동화된 빌드 및 테스트 인프라를 통과해야 합니다.
*   **회귀 테스트:** 이전 버전에서 잘 작동하던 기능이 망가지지 않았는지 확인합니다.

---
*참고: 이 프로세스는 프로덕션 환경에서 Yocto Project를 사용하는 기업과 개발자에게 높은 신뢰성을 제공하기 위해 설계되었습니다.*
