# Yocto Project 메이저 릴리스 코드명 (Major Release Codenames)

Yocto Project의 각 메이저 릴리스는 버전 번호(예: 4.0, 5.0) 외에도 고유한 **코드명(Codename)**을 가집니다. 이 코드명은 프로젝트 생태계에서 릴리스를 식별하고 소통하는 데 핵심적인 역할을 합니다.

## 1. 코드명 명명 규칙

Yocto Project의 코드명은 일관된 테마를 따릅니다.

*   **테마:** 주로 **영국의 지명**에서 유래합니다. 특히 호수 지방(Lake District)이나 국립공원 내의 산봉우리, 고개(Pass), 랜드마크 등의 이름을 사용합니다.
*   **예시:** *Kirkstone* (Pass), *Dunfell* (Great Dun Fell), *Scarthgap* (Pass).

## 2. 코드명의 용도

코드명은 단순한 별칭이 아니라 기술적인 설정과 구성 관리에서 중요하게 사용됩니다.

*   **Git 브랜치:** Yocto Project의 소스 저장소(Poky 및 각종 레이어)에서 브랜치 이름으로 사용됩니다.
    *   예: `git checkout kirkstone`
*   **레이어 호환성:** `conf/layer.conf` 파일 내의 `LAYERSERIES_COMPAT` 변수에 사용되어, 레이어 간의 호환성을 보장합니다.
    *   예: `LAYERSERIES_COMPAT_meta-mylayer = "kirkstone"`
*   **문서 및 커뮤니케이션:** 개발자 간의 대화, 메일링 리스트, 버그 리포트에서 특정 버전을 지칭할 때 버전 번호보다 코드명을 더 자주 사용합니다.
