# Yocto Project 핵심 구성 조각 (Core Configuration Fragments)

Yocto Project는 `local.conf`를 직접 수정하는 대신, 검증된 설정을 빠르게 적용할 수 있도록 **Core Configuration Fragments**를 제공합니다. 이들은 주로 `bitbake-config-build` 도구를 통해 관리됩니다.

## 1. Yocto Project Fragments (`core/yocto`)

일반적인 개발자가 유용하게 사용할 수 있는 표준 설정 조각들입니다.

### `core/yocto/sstate-mirror-cdn`
*   **설명:** Yocto Project가 호스팅하는 공용 SState(Shared State) 미러 서버를 사용하도록 설정합니다.
*   **효과:** 빌드 시 미리 빌드된 아티팩트를 다운로드하여 빌드 시간을 획기적으로 단축할 수 있습니다. 네트워크 대역폭이 충분하다면 매우 권장되는 설정입니다.

### `core/yocto/root-login-with-empty-password`
*   **설명:** 타겟 이미지의 `root` 사용자가 비밀번호 없이 로그인할 수 있도록 설정합니다.
*   **효과:** `EXTRA_IMAGE_FEATURES`에 `debug-tweaks`를 추가하는 것과 유사하며, 개발 및 디버깅 단계에서 편리하게 접근할 수 있게 해줍니다. (보안상 프로덕션 이미지에는 사용하지 않아야 합니다.)

## 2. Yocto Project Autobuilder Fragments (`core/yocto-autobuilder`)

Yocto Project의 공식 CI(Continuous Integration) 시스템인 **Autobuilder**에서 사용하기 위해 정의된 조각들입니다. 일반 사용자가 직접 사용할 일은 드물지만, 프로젝트의 테스트 구성을 이해하는 데 도움이 됩니다.

### 주요 조각들
*   **`autobuilder`**: 오토빌더 환경을 위한 기본 설정.
*   **`autobuilder-resource-constraints`**: 빌드 리소스 제한 설정.
*   **`multilib-mips64-n32`**: MIPS64 n32 멀티라이브러리 지원 설정.
*   **`multilib-x86-lib32`**: x86 32비트 멀티라이브러리 지원 설정.
*   **`multilib-x86-lib64`**: x86 64비트 멀티라이브러리 지원 설정.

---
*참고: 이 조각들은 `bitbake-config-build list-fragments` 명령어로 확인할 수 있습니다.*
