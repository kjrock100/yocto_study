# Yocto Project 핵심 내장 조각 (Core Built-in Fragments)

Yocto Project 빌드 시스템은 구성을 모듈화하기 위해 다양한 조각(Fragments)을 사용합니다. **Core Built-in Fragments**는 그 중 가장 기본이 되는 머신(Machine) 및 배포판(Distro) 구성 파일을 의미합니다.

## 1. 개요

이 조각들은 OpenEmbedded-Core(`meta/`) 레이어 내에 미리 정의되어 있으며, 빌드 환경의 기초를 형성합니다. 사용자는 `local.conf` 파일에서 `MACHINE` 및 `DISTRO` 변수를 설정함으로써 이 조각들을 활성화합니다.

## 2. 머신 구성 (`machine/`)

`meta/conf/machine/` 디렉토리에 위치한 `.conf` 파일들입니다. 타겟 하드웨어 플랫폼을 정의합니다.

*   **주요 설정 항목:**
    *   `TARGET_ARCH`: 타겟 시스템의 CPU 아키텍처 (예: x86_64, arm).
    *   `PREFERRED_PROVIDER_virtual/kernel`: 사용할 리눅스 커널 레시피.
    *   `MACHINE_FEATURES`: 하드웨어 기능 목록 (예: wifi, bluetooth, screen).
    *   `SERIAL_CONSOLES`: 시리얼 콘솔 설정.

*   **예시:** `qemux86-64.conf`, `qemuarm.conf`

## 3. 배포판 구성 (`distro/`)

`meta/conf/distro/` 디렉토리에 위치한 `.conf` 파일들입니다. 소프트웨어 배포판의 정책(Policy)을 정의합니다.

*   **주요 설정 항목:**
    *   `DISTRO_NAME` 및 `DISTRO_VERSION`: 배포판 이름과 버전.
    *   `TCMODE`: 툴체인 모드 (기본값: default).
    *   `TCLIBC`: C 라이브러리 선택 (예: glibc, musl).
    *   `INIT_MANAGER`: 초기화 시스템 선택 (예: systemd, sysvinit).
    *   `PACKAGE_CLASSES`: 패키지 형식 (예: package_rpm, package_deb).

*   **예시:** `poky.conf`, `poky-tiny.conf`

---
*참고: 이 문서는 Yocto Project 참조 매뉴얼의 Core Built-in Fragments 섹션을 기반으로 작성되었습니다.*
