# Yocto Project 타겟 장비에 소프트웨어 배포 (Deploying Your Software on the Target Machine)

Yocto Project에서 `devtool deploy-target` 명령어를 사용하여 워크스페이스에서 빌드한 레시피의 결과물을 타겟 장비에 배포하는 방법에 대해 설명합니다.

## 1. 개요

`devtool build`를 통해 레시피를 빌드한 후, 전체 이미지를 다시 굽지 않고도 변경된 패키지 파일들을 타겟 장비에 직접 전송하여 빠르게 테스트할 수 있습니다. 이 과정은 SSH를 통해 이루어집니다.

## 2. 사전 요구 사항

*   타겟 장비가 부팅되어 있어야 하며, 네트워크에 연결되어 있어야 합니다.
*   타겟 장비에 SSH 서버가 실행 중이어야 합니다 (대부분의 Yocto 개발용 이미지에는 `openssh`나 `dropbear`가 포함되어 있습니다).
*   호스트 머신에서 타겟 장비로 SSH 접속이 가능해야 합니다.

## 3. 기본 사용법

```bash
devtool deploy-target [options] <recipe> <target>
```

*   **`<recipe>`**: 배포할 레시피의 이름입니다.
*   **`<target>`**: 타겟 장비의 주소입니다. `user@ip` 형식으로 지정합니다 (예: `root@192.168.1.10`). 사용자를 생략하면 기본적으로 `root`가 사용될 수 있습니다.

## 4. 주요 옵션

### 4.1 상태 확인 (`--show-status`, `-s`)
현재 타겟에 배포된 파일들의 상태를 보여줍니다.

```bash
devtool deploy-target -s my-recipe root@192.168.1.10
```

### 4.2 드라이 런 (`--dry-run`, `-n`)
실제로 파일을 복사하지 않고, 어떤 작업이 수행될지 미리 확인합니다.

```bash
devtool deploy-target -n my-recipe root@192.168.1.10
```

## 5. 작동 방식

1.  `devtool`은 지정된 레시피의 빌드 결과물을 확인합니다.
2.  SSH를 통해 타겟 장비에 접속합니다.
3.  파일들을 타겟 장비의 루트 파일시스템(`/`)에 복사하여 설치합니다.

---
*참고: 이 명령은 패키지 매니저를 통하지 않고 파일을 직접 복사하므로, 테스트가 끝나면 `devtool undeploy-target`을 사용하여 정리하는 것이 좋습니다.*
