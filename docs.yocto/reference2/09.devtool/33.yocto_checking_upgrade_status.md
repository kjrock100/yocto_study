# Yocto Project 레시피 업그레이드 상태 확인 (Checking on the Upgrade Status of a Recipe)

Yocto Project에서 `devtool check-upgrade-status` 명령어를 사용하여 레시피의 업스트림(Upstream) 버전을 확인하고, 업데이트가 필요한지 파악하는 방법에 대해 설명합니다.

## 1. 개요

유지 관리 중인 레시피의 소스 코드가 업스트림에서 업데이트되었는지 주기적으로 확인하는 것은 중요합니다. `devtool`은 레시피에 정의된 정보를 바탕으로 업스트림 저장소나 다운로드 페이지를 확인하여 최신 버전을 감지할 수 있습니다.

## 2. 기본 사용법

```bash
devtool check-upgrade-status [recipe]
```

*   **`[recipe]`**: 상태를 확인할 특정 레시피 이름입니다. 생략하면 모든 레시피를 확인합니다.

## 3. 결과 확인

명령을 실행하면 다음과 같은 정보가 테이블 형태로 출력됩니다.

*   **Recipe Name**: 레시피 이름
*   **Current Version**: 현재 Yocto 빌드 시스템에 정의된 버전
*   **Latest Version**: 업스트림에서 감지된 최신 버전
*   **Status**: 상태 (예: `MATCH`, `UPDATE`, `UNKNOWN`)

### 상태 코드 의미
*   **MATCH**: 현재 버전이 최신 버전과 일치합니다.
*   **UPDATE**: 더 새로운 버전이 발견되었습니다. 업그레이드가 필요합니다.
*   **UNKNOWN**: 업스트림 버전을 확인할 수 없습니다. (설정 부족 또는 네트워크 문제 등)

## 4. 업스트림 확인 설정 (Configuration)

`devtool`이 업스트림 버전을 올바르게 감지하려면 레시피 내에 적절한 설정이 필요할 수 있습니다. 대부분의 경우 `SRC_URI`를 통해 자동으로 추측하지만, 명시적인 설정이 필요한 경우도 있습니다.

*   **`UPSTREAM_CHECK_URI`**: 버전을 확인할 URL입니다.
*   **`UPSTREAM_CHECK_REGEX`**: 버전 번호를 추출하기 위한 정규 표현식입니다.
*   **`UPSTREAM_CHECK_GITTAGREGEX`**: Git 저장소를 사용하는 경우 태그에서 버전을 추출하기 위한 정규 표현식입니다.

*참고: 이 기능은 `distrodata` 클래스의 기능을 활용합니다.*
