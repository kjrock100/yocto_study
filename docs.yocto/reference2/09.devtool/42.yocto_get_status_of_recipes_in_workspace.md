# Yocto Project 워크스페이스 레시피 상태 확인 (Get the Status of the Recipes in Your Workspace)

Yocto Project에서 `devtool status` 명령어를 사용하여 현재 워크스페이스에 있는 레시피들의 상태를 확인하는 방법에 대해 설명합니다.

## 1. 개요

`devtool modify`, `devtool add`, `devtool upgrade` 등의 명령을 사용하면 레시피가 워크스페이스 레이어(`build/workspace`)에 추가됩니다. `devtool status` 명령은 현재 워크스페이스에서 작업 중인 모든 레시피의 목록과 관련 경로 정보를 제공합니다.

## 2. 기본 사용법

```bash
devtool status
```

이 명령은 별도의 옵션 없이 실행됩니다.

## 3. 출력 정보

명령을 실행하면 각 레시피에 대해 다음 정보가 출력됩니다.

*   **레시피 이름 (Recipe Name)**
*   **소스 트리 경로 (Source Tree Path):** 소스 코드가 추출되어 있는 디렉토리 경로입니다.
*   **레시피 파일 경로 (Recipe File Path):** 워크스페이스 내의 `.bbappend` 파일 또는 `.bb` 파일의 경로입니다.

## 4. 사용 예시

```bash
$ devtool status
bash: /home/user/poky/build/workspace/sources/bash /home/user/poky/build/workspace/appends/bash_%.bbappend
my-app: /home/user/poky/build/workspace/sources/my-app /home/user/poky/build/workspace/recipes/my-app/my-app_git.bb
```

위 예시에서:
*   `bash`는 기존 레시피를 수정 중(`modify`)이므로 `appends` 디렉토리의 `.bbappend` 파일을 참조합니다.
*   `my-app`은 새로 추가된 레시피(`add`)이므로 `recipes` 디렉토리의 `.bb` 파일을 참조합니다.

---
*참고: 워크스페이스에 레시피가 없다면 아무것도 출력되지 않습니다.*
