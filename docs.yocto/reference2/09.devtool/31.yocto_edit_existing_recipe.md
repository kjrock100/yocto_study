# Yocto Project 기존 레시피 편집 (Edit an Existing Recipe)

Yocto Project에서 `devtool edit-recipe` 명령어를 사용하여 레시피 파일(`.bb`) 자체를 편집하는 방법에 대해 설명합니다.

## 1. 개요

`devtool edit-recipe` 명령은 지정된 레시피 파일을 기본 텍스트 에디터에서 엽니다. 이는 소스 코드를 수정하는 것이 아니라, 레시피의 메타데이터(변수, 의존성, 작업 정의 등)를 빠르게 수정해야 할 때 유용합니다.

## 2. 기본 사용법

```bash
devtool edit-recipe <recipe>
```

*   **`<recipe>`**: 편집할 레시피의 이름입니다.

## 3. 작동 방식

이 명령을 실행하면 `devtool`은 해당 레시피 파일의 위치를 찾아 `EDITOR` 환경 변수에 설정된 에디터로 파일을 엽니다.

*   만약 `EDITOR` 환경 변수가 설정되어 있지 않다면, 기본적으로 `vi` 등을 시도할 수 있습니다.
*   워크스페이스(`workspace`)에 있는 레시피뿐만 아니라, 다른 레이어에 있는 레시피도 편집할 수 있습니다. 하지만 워크스페이스 외부의 레시피를 직접 수정하는 것은 권장되지 않으며(레이어 관리가 복잡해짐), 주로 워크스페이스 내의 레시피를 다듬을 때 사용합니다.

## 4. `devtool modify`와의 차이점

*   **`devtool modify`**: 소스 코드(`src`)를 추출하여 수정하고 빌드하는 데 초점을 맞춥니다. C 코드나 Python 스크립트 등 실제 프로그램 로직을 고칠 때 사용합니다.
*   **`devtool edit-recipe`**: 레시피 파일(`.bb`)을 수정하는 데 초점을 맞춥니다. `SRC_URI`, `DEPENDS`, `EXTRA_OECONF` 등 빌드 설정이나 메타데이터를 변경할 때 사용합니다.

## 5. 사용 예시

`bash` 레시피의 빌드 옵션을 변경하고 싶다고 가정해 봅시다.

1.  에디터 설정 (선택 사항):
    ```bash
    export EDITOR=nano
    ```
2.  레시피 편집:
    ```bash
    devtool edit-recipe bash
    ```
3.  에디터가 열리면 필요한 변수(예: `EXTRA_OECONF`)를 수정하고 저장합니다.

---
*참고: 워크스페이스에 없는 레시피를 수정하면 원본 레이어의 파일이 직접 변경되므로 주의해야 합니다. 안전한 작업을 위해 먼저 `devtool modify`를 통해 워크스페이스로 가져오는 것이 좋습니다.*
