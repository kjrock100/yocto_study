# Yocto Project 소개

## 1. Yocto Project란 무엇인가?

Yocto Project는 임베디드 리눅스 배포판 그 자체가 아니라, **사용자 맞춤형 임베디드 리눅스 배포판을 생성하기 위한 오픈 소스 협업 프로젝트**입니다. 하드웨어 아키텍처에 구애받지 않고 커스텀 리눅스 시스템을 구축할 수 있는 템플릿, 도구, 방법론을 제공합니다.

> "It's not an embedded Linux distribution, it creates a custom one for you."

## 2. 왜 Yocto를 사용하는가?

*   **맞춤형 구성:** 프로젝트의 스펙에 정확히 맞춰 리눅스를 구성할 수 있습니다.
*   **최적화:** 필요한 패키지만 포함시켜 이미지 크기를 최소화하고 성능을 최적화할 수 있습니다.
*   **유연성:** 다양한 하드웨어 아키텍처(x86, ARM, MIPS, PowerPC 등)를 지원합니다.
*   **재현성:** 소스 코드 기반의 빌드 시스템을 통해 언제든지 동일한 결과물을 재생성할 수 있습니다.

## 3. 주요 구성 요소

### Poky (포키)
Yocto Project의 **레퍼런스 배포판**입니다. 빌드 시스템(BitBake, OpenEmbedded-Core)과 메타데이터 세트를 포함하고 있어, 사용자가 즉시 빌드를 시작할 수 있는 환경을 제공합니다.

### BitBake (비트베이크)
Yocto Project의 **빌드 엔진**입니다. 레시피(Recipe) 파일들을 읽어 의존성을 파악하고, 소스 코드를 가져와 빌드 및 패키징 작업을 수행합니다. `make`와 유사하지만 훨씬 강력하고 복잡한 기능을 수행합니다.

### OpenEmbedded-Core (OE-Core)
빌드 시스템의 핵심 메타데이터 집합입니다. 기본적인 레시피, 클래스, 설정 파일들이 포함되어 있습니다.

### Metadata (메타데이터)
빌드 대상을 정의하는 파일들입니다.
*   **Recipes (.bb):** 소프트웨어 패키지를 빌드하는 방법(소스 위치, 패치, 컴파일 옵션 등)을 정의합니다.
*   **Configuration (.conf):** 전역 설정, 머신 설정, 배포판 설정 등을 정의합니다.

## 4. 기본 워크플로우

1.  **소스 다운로드:** Poky 및 필요한 레이어(Layer)를 다운로드합니다.
2.  **환경 설정:** `oe-init-build-env` 스크립트를 실행하여 빌드 환경을 초기화합니다.
3.  **설정 변경:** `conf/local.conf` 파일에서 타겟 머신(Machine), 다운로드 경로 등을 설정합니다.
4.  **빌드 실행:** `bitbake` 명령어를 사용하여 이미지(예: `core-image-minimal`)를 빌드합니다.
5.  **이미지 배포:** 생성된 이미지를 타겟 하드웨어에 플래싱하거나 QEMU로 실행합니다.

## 5. 지원 하드웨어

Yocto Project는 QEMU 에뮬레이터뿐만 아니라 다양한 실제 하드웨어를 지원합니다. 대표적으로 BeagleBone Black, Raspberry Pi, Intel 기반 보드 등 다양한 BSP(Board Support Package) 레이어가 제공됩니다.
