# local.conf를 사용한 이미지 커스터마이징 (Customizing Images Using local.conf)

Yocto Project에서 이미지를 커스터마이징하는 가장 빠르고 간단한 방법은 빌드 디렉토리의 `conf/local.conf` 파일을 수정하는 것입니다. 이 방법은 개발 단계에서 패키지를 빠르게 추가하거나 설정을 테스트할 때 유용합니다.

## 1. 패키지 추가 (Adding Packages)

이미지에 추가 패키지를 설치하려면 `IMAGE_INSTALL:append` 변수를 사용합니다.

### 1.1. 설정 방법

`conf/local.conf` 파일을 열고 다음과 같이 라인을 추가합니다.

```bash
IMAGE_INSTALL:append = " package-name"
```

**주의사항**:

- 값의 시작 부분에 **공백(space)**이 반드시 포함되어야 합니다. 그렇지 않으면 이전 값과 붙어서 해석되어 빌드 오류가 발생할 수 있습니다.
- 예: `strace` 패키지를 추가하는 경우

  ```bash
  IMAGE_INSTALL:append = " strace"
  ```

### 1.2. 여러 패키지 추가

여러 패키지를 추가할 때는 공백으로 구분하여 나열합니다.

```bash
IMAGE_INSTALL:append = " strace htop vim"
```

## 2. 장점과 단점

### 2.1. 장점

- **간편함**: 별도의 레이어나 레시피를 생성할 필요가 없습니다.
- **신속함**: 설정 파일만 수정하면 되므로 테스트 주기가 빠릅니다.

### 2.2. 단점

- **전역 적용**: 해당 빌드 디렉토리에서 생성되는 **모든** 이미지에 적용됩니다. 예를 들어, `core-image-minimal`과 `core-image-sato`를 모두 빌드할 경우 두 이미지 모두에 패키지가 추가됩니다.
- **유지보수**: 장기적인 프로젝트 관리나 배포용 이미지 생성 시에는 전용 이미지 레시피를 만드는 것이 더 체계적입니다.
