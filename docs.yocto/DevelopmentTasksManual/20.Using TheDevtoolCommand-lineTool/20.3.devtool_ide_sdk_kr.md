# devtool ide-sdk를 사용하여 IDE 구성 및 SDK 부트스트랩 (devtool ide-sdk configures IDEs and bootstraps SDKs)

`devtool ide-sdk` 명령은 Yocto Project 환경에서 개발자가 친숙한 통합 개발 환경(IDE)을 사용하여 애플리케이션을 개발, 빌드 및 디버깅할 수 있도록 돕습니다. 이 명령은 필요한 SDK(Software Development Kit)를 자동으로 생성(부트스트랩)하고, IDE에 맞는 설정 파일을 생성해줍니다.

## 1. 개요

임베디드 리눅스 개발 시 크로스 컴파일 환경 설정과 헤더 파일 경로 설정은 번거로운 작업입니다. `devtool ide-sdk`는 작업 공간(Workspace)에 있는 레시피를 위해 이 과정을 자동화합니다.

* **SDK 준비**: 해당 이미지를 빌드하는 데 필요한 툴체인과 라이브러리가 포함된 SDK를 생성하거나 준비합니다.
* **IDE 설정 생성**: VSCode 등의 IDE가 크로스 컴파일러와 헤더 경로를 인식할 수 있도록 설정 파일(예: `c_cpp_properties.json`, `launch.json`)을 생성합니다.

## 2. 기본 구문

```bash
devtool ide-sdk [options] <recipename> <target>
```

* **recipename**: 작업 공간에서 수정 중인 레시피의 이름입니다 (예: `my-app`).
* **target**: 설정을 생성할 IDE 또는 대상입니다. (예: `code` (VSCode), `cmake`, `none`).

## 3. 사용 가능한 타겟 (Target)

* **code**: Visual Studio Code (VSCode)용 설정을 생성합니다.
* **cmake**: CMake 프로젝트를 위한 툴체인 파일 및 프리셋을 준비합니다.
* **none**: SDK만 준비하고 별도의 IDE 설정은 생성하지 않습니다.

## 4. 작업 흐름 (Workflow)

VSCode를 사용하여 `my-app`이라는 애플리케이션을 개발하는 일반적인 절차는 다음과 같습니다.

### 4.1 레시피 수정 모드 진입

먼저 `devtool modify`를 통해 소스 코드를 작업 공간으로 가져옵니다.

```bash
devtool modify my-app
```

### 4.2 IDE 설정 및 SDK 생성

`devtool ide-sdk`를 실행하여 VSCode용 설정을 생성합니다. 이 과정에서 필요한 SDK가 빌드될 수 있어 시간이 소요될 수 있습니다.

```bash
devtool ide-sdk my-app code
```

### 4.3 IDE 실행

명령이 완료되면 작업 공간의 소스 디렉토리에서 VSCode를 엽니다.

```bash
code workspace/sources/my-app
```

VSCode가 열리면 자동으로 다음 기능들이 활성화됩니다:

* **IntelliSense**: 크로스 컴파일용 헤더 파일 경로가 인식되어 코드 자동 완성이 작동합니다.
* **디버깅**: `launch.json`이 구성되어 있어, GDB를 통한 원격 디버깅(QEMU 또는 타겟 보드)이 가능합니다.
