# Yocto Project 소스 파일 위치 확인 (Locating Yocto Project Source Files)

Yocto Project 개발을 시작하려면 먼저 소스 코드를 확보해야 합니다. 이 문서는 소스 저장소, 아카이브, 다운로드 페이지를 통해 Yocto Project 소스 파일을 찾는 방법을 설명합니다.

## 1. 소스 저장소 접근 (Accessing Source Repositories)
Yocto Project는 주로 Git을 사용하여 소스 코드를 관리합니다. 개발자는 업스트림 저장소에 접근하여 최신 소스 코드나 특정 브랜치를 복제(Clone)할 수 있습니다.

- **웹 인터페이스**: [http://git.yoctoproject.org/](http://git.yoctoproject.org/)
  - Yocto Project의 모든 공식 저장소(Poky, BitBake, 메타데이터 레이어 등)를 탐색할 수 있습니다.
  - 특정 저장소를 클릭하면 커밋 로그, 브랜치, 태그 정보를 확인할 수 있습니다.
- **주요 저장소**:
  - `poky`: Yocto Project의 참조 배포판 및 빌드 시스템.
  - `openembedded-core`: 핵심 메타데이터.
  - `bitbake`: 빌드 엔진.

## 2. 소스 아카이브 접근 (Accessing Source Archives)
Git 저장소의 전체 히스토리가 필요하지 않거나, 방화벽 문제로 Git 프로토콜을 사용할 수 없는 경우 소스 아카이브(Tarball)를 사용할 수 있습니다.

- **스냅샷**: Git 웹 인터페이스의 각 저장소 페이지에서 특정 커밋이나 태그에 대한 'snapshot' 링크를 제공합니다. 이를 통해 해당 시점의 소스 코드를 `.tar.gz` 등의 압축 파일로 다운로드할 수 있습니다.
- **장점**: 다운로드 속도가 빠르고 설정이 간편합니다.
- **단점**: Git의 버전 관리 기능(업데이트, 패치 관리 등)을 직접 사용할 수 없습니다.

## 3. 다운로드 페이지 사용 (Using the Downloads Page)
Yocto Project 공식 웹사이트는 릴리스된 버전의 소프트웨어를 쉽게 찾을 수 있는 다운로드 페이지를 제공합니다.

- **URL**: https://www.yoctoproject.org/software-overview/downloads/
- **제공되는 파일**:
  - **Poky 릴리스**: 테스트가 완료된 안정적인 버전의 빌드 시스템.
  - **툴체인**: 애플리케이션 개발을 위한 크로스 컴파일 툴체인 스크립트.
  - **이미지**: QEMU 등에서 실행 가능한 미리 빌드된 이미지.
