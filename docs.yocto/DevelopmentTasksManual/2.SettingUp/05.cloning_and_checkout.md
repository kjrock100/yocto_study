# Yocto Project 브랜치 복제 및 체크아웃 (Cloning and Checking Out Branches)

Yocto Project 개발을 시작하기 위해 소스 코드를 로컬 머신으로 가져오는 과정입니다. 이 문서는 `poky` 저장소를 복제(Clone)하고 특정 브랜치나 태그를 체크아웃하는 방법을 설명합니다.

## 1. Poky 저장소 복제 (Cloning the poky Repository)
Yocto Project의 참조 배포판인 Poky의 소스 코드를 가져오기 위해 Git을 사용합니다.

- **명령어**:
  ```bash
  git clone git://git.yoctoproject.org/poky
  cd poky
  ```
  이 명령은 현재 디렉토리에 `poky`라는 하위 디렉토리를 생성하고 최신 소스 코드를 다운로드합니다.

## 2. 브랜치 체크아웃 (Checking Out by Branch in Poky)
Yocto Project는 6개월마다 새로운 릴리스를 발표하며, 각 릴리스는 고유한 코드명(예: `scarthgap`, `nanbield`, `kirkstone` 등)을 가진 브랜치로 관리됩니다. 안정적인 개발을 위해 특정 릴리스 브랜치를 사용하는 것이 좋습니다.

- **브랜치 목록 확인**:
  ```bash
  git branch -a
  ```
- **특정 브랜치 체크아웃** (예: `scarthgap`):
  ```bash
  git checkout -t origin/scarthgap -b my-scarthgap
  ```
  또는 간단히:
  ```bash
  git checkout scarthgap
  ```
  이렇게 하면 해당 릴리스의 최신 상태로 작업 디렉토리가 업데이트됩니다.

## 3. 태그 체크아웃 (Checking Out by Tag in Poky)
특정 시점의 릴리스 버전(예: `yocto-5.0`, `yocto-4.0.1`)으로 고정하여 빌드하고 싶다면 태그를 체크아웃할 수 있습니다.

- **태그 목록 확인**:
  ```bash
  git tag
  ```
- **특정 태그 체크아웃** (예: `yocto-5.0`):
  ```bash
  git checkout tags/yocto-5.0 -b my-yocto-5.0
  ```
  태그를 기반으로 새로운 로컬 브랜치(`my-yocto-5.0`)를 생성하여 작업하는 것이 안전합니다.
