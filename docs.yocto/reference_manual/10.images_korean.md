# 이미지

OpenEmbedded 빌드 시스템은 다양한 요구를 충족하기 위해 여러 가지 예제 이미지를 제공합니다. `bitbake` 명령을 실행할 때 원하는 이미지 유형에 대한 빌드를 본질적으로 시작하는 "최상위" 레시피를 제공합니다.

> **참고:**
> GNU General Public License Version 3 (GPLv3), GNU Lesser General Public License Version 3 (LGPLv3) 및 GNU Affero General Public License Version 3 (AGPL-3.0) 구성 요소 없이 이미지를 빌드하는 것은 core-image-minimal 이미지에 대해서만 테스트됩니다. 또한 이미지를 빌드하고 GPLv3 및 유사하게 라이선스된 구성 요소를 포함하지 않는지 확인하려면 BitBake 명령을 사용하여 이미지를 빌드하기 전에 이미지 레시피 파일에서 다음 변경을 수행해야 합니다:
>
> ```
> INCOMPATIBLE_LICENSE = "GPL-3.0* LGPL-3.0*"
> ```
>
> 또는 라이선스 제한이 적용되어야 하는 모든 이미지에 대해 라인을 반복하고 조정하여 `local.conf` 파일을 조정할 수 있습니다:
>
> ```
> INCOMPATIBLE_LICENSE:pn-your-image-name = "GPL-3.0* LGPL-3.0*"
> ```

`poky` Git 저장소 내에서 다음 명령을 사용하여 이미지 레시피 파일을 포함하는 소스 디렉토리 내의 디렉토리 목록을 표시할 수 있습니다:

```
$ ls meta*/recipes*/images/*.bb
```

지원되는 레시피 목록:

## build-appliance-image

모든 빌드 시스템 자체뿐만 아니라 빌드 시스템을 사용하여 빌드를 실행하는 데 필요한 모든 조각을 포함하는 예제 가상 머신입니다. VMware Player 또는 VMware Workstation을 사용하여 이미지를 부팅하고 실행할 수 있습니다. 이 이미지에 대한 자세한 정보는 Yocto Project 웹사이트의 Build Appliance 페이지를 참조하세요.

## core-image-base

타겟 디바이스 하드웨어를 완벽하게 지원하는 콘솔 전용 이미지입니다.

## core-image-full-cmdline

더 완전한 Linux 시스템 기능이 설치된 콘솔 전용 이미지입니다.

## core-image-minimal

디바이스가 부팅할 수 있도록 하는 작은 이미지입니다.

## core-image-minimal-dev

호스트를 사용한 개발 작업에 적합한 `core-image-minimal` 이미지입니다. 이미지는 호스트 개발 환경에서 사용할 수 있는 헤더 및 라이브러리를 포함합니다.

## core-image-minimal-initramfs

커널의 일부로 Minimal RAM 기반 초기 루트 파일시스템(Initramfs)을 갖는 `core-image-minimal` 이미지로, 시스템이 첫 번째 "init" 프로그램을 더 효율적으로 찾을 수 있습니다. Initramfs 이미지로 작업할 때 도움이 되는 추가 정보는 PACKAGE_INSTALL 변수를 참조하세요.

## core-image-minimal-mtdutils

Minimal MTD Utilities를 지원하는 `core-image-minimal` 이미지로, 사용자가 MTD 서브시스템과 상호 작용하여 플래시 디바이스에 대한 작업을 수행할 수 있습니다.

## core-image-rt

실시간 사용에 적합한 실시간 테스트 제품군 및 도구와 함께 `core-image-minimal` 이미지입니다.

## core-image-rt-sdk

크로스 툴체인의 모든 것을 포함하는 `core-image-rt` 이미지입니다. 이미지는 또한 완전한 독립 실행형 SDK를 형성하는 개발 헤더 및 라이브러리를 포함하며 타겟을 사용한 개발에 적합합니다.

## core-image-sato

Sato 지원이 있는 이미지로, 모바일 디바이스에 잘 작동하는 모바일 환경 및 시각적 스타일입니다. 이미지는 Sato 테마 및 터미널, 편집기, 파일 관리자, 미디어 플레이어 등과 같은 응용 프로그램이 있는 X11을 지원합니다.

## core-image-sato-dev

호스트를 사용한 개발에 적합한 `core-image-sato` 이미지입니다. 이미지는 디바이스 자체에서 응용 프로그램을 빌드하는 데 필요한 라이브러리, 테스트 및 프로파일링 도구, 디버그 심볼을 포함합니다. 이 이미지는 이전에 `core-image-sdk`였습니다.

## core-image-sato-sdk

크로스 툴체인의 모든 것을 포함하는 `core-image-sato` 이미지입니다. 이미지는 또한 완전한 독립 실행형 SDK를 형성하는 개발 헤더 및 라이브러리를 포함하며 타겟을 사용한 개발에 적합합니다.

## core-image-testmaster

자동화된 런타임 테스팅에 사용하도록 설계된 "컨트롤러" 이미지입니다. 별도의 파티션에 배포되는 "알려진 좋은" 이미지를 제공하므로 이미지로 부팅하여 테스트할 두 번째 이미지를 배포할 수 있습니다. Yocto Project 테스트 환경 설명서의 "Performing Automated Runtime Testing" 섹션에서 런타임 테스팅에 대한 자세한 정보를 찾을 수 있습니다.

## core-image-testmaster-initramfs

`core-image-testmaster` 이미지와 함께 사용하도록 맞춤식된 RAM 기반 초기 루트 파일시스템(Initramfs) 이미지입니다.

## core-image-weston

터미널이 있는 매우 기본적인 Wayland 이미지입니다. 이 이미지는 Wayland 프로토콜 라이브러리 및 참조 Weston 컴포지터를 제공합니다. 자세한 정보는 Yocto Project 개발 작업 설명서의 "Using Wayland and Weston" 섹션을 참조하세요.

## core-image-x11

터미널이 있는 매우 기본적인 X11 이미지입니다.
