# Yocto Project bitbake-config-build 도구

`bitbake-config-build`는 Yocto Project 빌드 환경 설정(`local.conf`)을 관리하기 위한 커맨드라인 도구입니다. 이 도구를 사용하면 미리 정의된 **설정 조각(Configuration Fragments)**을 통해 복잡한 설정을 쉽게 적용하거나 해제할 수 있습니다.

## 1. 개요

기존에는 `local.conf` 파일을 직접 편집하여 설정을 변경해야 했지만, `bitbake-config-build`를 사용하면 모듈화된 설정 조각을 활성화하거나 비활성화하는 방식으로 빌드 환경을 제어할 수 있습니다. 이는 설정의 재사용성을 높이고 오류를 줄여줍니다.

## 2. 주요 명령어 (Quick Reference)

이 도구는 빌드 환경이 초기화된 상태(`source oe-init-build-env` 실행 후)에서 사용할 수 있습니다.

### `list-fragments`
사용 가능한 모든 설정 조각의 목록을 표시합니다.
```bash
bitbake-config-build list-fragments
```

### `show-fragment <fragment>`
특정 설정 조각의 내용을 확인합니다.
```bash
bitbake-config-build show-fragment core/yocto/sstate-mirror-cdn
```

### `enable-fragment <fragment>`
특정 설정 조각을 활성화하여 현재 빌드 설정에 적용합니다.
```bash
bitbake-config-build enable-fragment core/yocto/sstate-mirror-cdn
```

### `disable-fragment <fragment>`
활성화된 설정 조각을 비활성화합니다.
```bash
bitbake-config-build disable-fragment core/yocto/sstate-mirror-cdn
```

### `disable-all-fragments`
현재 활성화된 모든 설정 조각을 비활성화합니다.
```bash
bitbake-config-build disable-all-fragments
```

## 3. 사용 예시

### SState 미러 CDN 사용
빌드 속도를 높이기 위해 Yocto Project에서 제공하는 CDN 기반 SState 미러를 사용하고 싶다면 다음과 같이 실행합니다.

```bash
$ bitbake-config-build enable-fragment core/yocto/sstate-mirror-cdn
```

### 루트 비밀번호 제거 (개발용)
개발 중 디버깅 편의를 위해 루트 사용자가 비밀번호 없이 로그인하도록 설정합니다.

```bash
$ bitbake-config-build enable-fragment core/yocto/root-login-with-empty-password
```
