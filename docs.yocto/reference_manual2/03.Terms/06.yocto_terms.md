# Yocto Project 주요 용어 (Terms)

Yocto Project를 이해하고 사용하기 위해 알아야 할 주요 용어들을 정리했습니다.

## 1. 핵심 구성 요소 (Core Components)

### BitBake
Yocto Project의 **빌드 엔진**입니다. 쉘 스크립트와 Python으로 작성되었으며, 레시피(Recipe) 파일을 읽어 의존성을 파악하고, 소스 코드를 가져와 빌드 및 패키징 작업을 수행합니다. `make`와 유사하지만 훨씬 강력하고 복잡한 기능을 수행합니다.

### Poky
Yocto Project의 **레퍼런스 배포판(Reference Distribution)**입니다. 빌드 시스템(BitBake, OpenEmbedded-Core)과 메타데이터 세트를 포함하고 있어, 사용자가 즉시 빌드를 시작할 수 있는 환경을 제공합니다.

### OpenEmbedded-Core (OE-Core)
빌드 시스템의 **핵심 메타데이터 집합**입니다. 기본적인 레시피, 클래스, 설정 파일들이 포함되어 있으며, Poky의 기반이 됩니다.

## 2. 메타데이터 및 구조 (Metadata & Structure)

### Recipe (.bb)
소프트웨어 패키지를 빌드하는 방법을 정의한 파일입니다. 소스 코드의 위치, 적용할 패치, 컴파일 옵션, 의존성 등이 기술되어 있습니다.

### Layer (레이어)
관련된 메타데이터(레시피, 설정 파일 등)의 집합입니다. Yocto Project는 레이어 구조를 통해 모듈화되어 있어, `meta-intel`, `meta-raspberrypi`와 같이 특정 하드웨어나 기능을 위한 레이어를 쉽게 추가하거나 제거할 수 있습니다.

### Metadata
빌드 대상을 정의하는 모든 파일(레시피, 설정 파일, 클래스 등)을 통칭합니다.

### Configuration (.conf)
전역 설정(`local.conf`), 머신 설정(`machine.conf`), 배포판 설정(`distro.conf`) 등 빌드 환경과 대상을 정의하는 설정 파일입니다.

## 3. 빌드 환경 (Build Environment)

### Build Directory
빌드 작업이 실제로 수행되고 결과물이 저장되는 디렉토리입니다. `oe-init-build-env` 스크립트를 실행하면 생성되며, `conf/local.conf` 파일이 이곳에 위치합니다.

### Source Directory
Poky 및 다운로드한 레이어들이 위치한 디렉토리입니다.

### Cross-Development Toolchain
호스트 시스템(예: x86 PC)에서 타겟 시스템(예: ARM 보드)용 실행 파일을 생성하기 위한 컴파일러, 링커, 라이브러리 등의 도구 모음입니다.

## 4. 기타 (Others)

### BSP (Board Support Package)
특정 하드웨어 보드를 지원하기 위한 메타데이터의 집합입니다. 커널 설정, 부트로더, 드라이버 등이 포함됩니다.

### Image
빌드 프로세스의 최종 결과물로, 타겟 디바이스에 플래싱하거나 실행할 수 있는 운영체제 이미지입니다.
