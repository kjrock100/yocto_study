# Yocto Project 클래스 (Classes)

Yocto Project에서 **클래스(Classes)**는 빌드 로직을 캡슐화하고 재사용하기 위한 메커니즘입니다. `.bbclass` 확장자를 가지며, 레시피 파일(`.bb`)이나 다른 클래스 파일에서 상속받아 공통된 기능을 쉽게 구현할 수 있게 해줍니다.

## 1. 클래스의 역할

*   **코드 재사용:** 여러 레시피에서 공통적으로 수행해야 하는 작업(예: `configure`, `compile`, `install`)을 정의해 두고, 이를 상속받아 중복을 줄입니다.
*   **추상화:** 복잡한 빌드 시스템(Autotools, CMake 등)의 세부 사항을 감추고, 간단한 변수 설정만으로 빌드가 가능하게 합니다.
*   **기능 확장:** 패키징, 사용자 추가, 시스템 서비스 등록 등 특정 기능을 레시피에 쉽게 추가할 수 있습니다.

## 2. 사용 방법

레시피 파일 내에서 `inherit` 키워드를 사용하여 클래스를 상속받습니다.

```bitbake
inherit autotools
inherit useradd
```

## 3. 주요 클래스 소개

Yocto Project(`meta/classes`)에는 수많은 클래스가 존재합니다. 그 중 자주 사용되는 핵심 클래스들은 다음과 같습니다.

### 빌드 시스템 관련
*   **`autotools`**: GNU Autotools(`configure`, `make`, `make install`)를 사용하는 소프트웨어를 빌드할 때 사용합니다.
*   **`cmake`**: CMake 빌드 시스템을 사용하는 소프트웨어를 위한 클래스입니다.
*   **`meson`**: Meson 빌드 시스템을 지원합니다.
*   **`python3native` / `setuptools3`**: Python 애플리케이션 및 모듈 빌드를 지원합니다.

### 커널 및 모듈 관련
*   **`kernel`**: 리눅스 커널을 빌드하기 위한 복잡한 로직을 포함하고 있습니다.
*   **`module`**: 리눅스 커널 모듈(Out-of-tree module)을 빌드할 때 사용합니다.

### 시스템 및 패키징 관련
*   **`base`**: 모든 레시피가 기본적으로 상속받는 클래스입니다. 기본적인 작업(`do_fetch`, `do_unpack` 등)이 정의되어 있습니다.
*   **`image`**: 루트 파일시스템 이미지를 생성하는 레시피에서 사용됩니다.
*   **`useradd`**: 빌드 시 시스템 사용자나 그룹을 추가해야 할 때 사용합니다.
*   **`systemd` / `update-rc.d`**: 시스템 서비스(Systemd 유닛 또는 SysVinit 스크립트)를 설치하고 활성화하는 데 사용됩니다.
*   **`package`**: 패키징 작업(`do_package`)을 처리합니다.

### 개발 및 디버깅
*   **`devshell`**: `bitbake -c devshell` 명령을 통해 크로스 컴파일 환경이 설정된 쉘을 열 수 있게 해줍니다.
*   **`rm_work`**: 빌드 완료 후 작업 디렉토리를 삭제하여 디스크 공간을 절약합니다 (`local.conf`에서 주로 설정).

### 기타
*   **`native`**: 호스트 시스템에서 실행될 도구를 빌드할 때 사용합니다.
*   **`nativesdk`**: SDK(Software Development Kit)에 포함될 도구를 빌드할 때 사용합니다.
*   **`allarch`**: 아키텍처에 독립적인 패키지(스크립트, 설정 파일 등)를 빌드할 때 사용하여 빌드 시간을 단축합니다.

---
*참고: 모든 클래스 파일은 `meta/classes` 디렉토리(또는 레이어의 `classes` 디렉토리)에 위치합니다.*
