# Yocto Project 레시피 빌드 (Building Your Recipe)

Yocto Project에서 `devtool build` 명령어를 사용하여 워크스페이스에 있는 레시피를 빌드하는 방법에 대해 설명합니다.

## 1. 개요

`devtool modify`나 `devtool add`를 통해 워크스페이스에 레시피를 설정하고 소스 코드를 수정한 후, 변경 사항이 올바르게 컴파일되고 패키징되는지 확인하기 위해 빌드를 수행해야 합니다. `devtool build` 명령은 전체 시스템 이미지를 다시 빌드하지 않고 해당 레시피만 빠르게 빌드할 수 있게 해줍니다.

## 2. 기본 사용법

```bash
devtool build [options] <recipe>
```

*   **`<recipe>`**: 빌드할 레시피의 이름입니다.

## 3. 주요 특징

*   **빠른 피드백:** 전체 이미지를 빌드하는 것보다 훨씬 빠르며, 수정한 소스 코드의 컴파일 오류를 즉시 확인할 수 있습니다.
*   **워크스페이스 연동:** 워크스페이스의 소스 트리(`build/workspace/sources/<recipe>`)를 사용하여 빌드를 수행합니다.
*   **작업 범위:** 주로 `do_compile`, `do_install`, `do_package` 등의 작업을 수행하여 패키지 파일(`.rpm`, `.ipk`, `.deb`)을 생성합니다.

## 4. 주요 옵션

### 4.1 병렬 빌드 비활성화 (`--disable-parallel-make`, `-s`)
빌드 중 경쟁 상태(Race Condition)가 의심되거나 로그를 순차적으로 확인하고 싶을 때 병렬 빌드를 끕니다.

```bash
devtool build -s my-recipe
```

## 5. 워크플로우 예시

1.  **소스 수정:** 워크스페이스의 소스 디렉토리에서 코드를 수정합니다.
2.  **빌드:** `devtool build my-recipe`를 실행합니다.
3.  **결과 확인:** 빌드가 성공하면 `devtool deploy-target`을 사용하여 타겟 장비에 배포하거나, 생성된 패키지를 확인합니다.

---
*참고: `devtool build`는 `bitbake -c build <recipe>`와 유사하지만, 워크스페이스 환경을 자동으로 처리한다는 점에서 차이가 있습니다.*
