# Yocto Project 레시피 초기화 (Resetting a Recipe)

Yocto Project에서 `devtool reset` 명령어를 사용하여 워크스페이스에서 작업 중인 레시피를 제거하고 원래 상태로 되돌리는 방법에 대해 설명합니다.

## 1. 개요

`devtool reset` 명령은 지정된 레시피를 워크스페이스 레이어(`build/workspace`)에서 제거합니다. 이 명령을 실행하면 빌드 시스템은 더 이상 워크스페이스의 로컬 소스 코드를 사용하지 않고, 원래 레이어에 정의된 레시피와 소스를 사용하게 됩니다.

## 2. 기본 사용법

```bash
devtool reset <recipe>
```

*   **`<recipe>`**: 초기화할 레시피의 이름입니다.

## 3. 주요 옵션

### 3.1 모든 레시피 초기화 (`--all`, `-a`)
워크스페이스에 있는 모든 레시피를 한 번에 초기화합니다.

```bash
devtool reset --all
```

### 3.2 소스 유지 (`--no-clean`, `-n`)
기본적으로 `devtool reset`은 워크스페이스에서 레시피 설정을 제거할 뿐만 아니라, `build/workspace/sources/<recipe>`에 있는 소스 트리도 삭제합니다. 만약 소스 코드는 남겨두고 레시피 설정만 워크스페이스에서 제거하고 싶다면 이 옵션을 사용합니다.

```bash
devtool reset --no-clean my-recipe
```

## 4. 사용 시나리오

*   **작업 취소:** 수정한 내용이 마음에 들지 않아 작업을 취소하고 원래 상태로 돌아가고 싶을 때.
*   **작업 완료 후 정리:** `devtool finish`를 사용하지 않고 단순히 워크스페이스를 비우고 싶을 때. (주의: 변경 사항이 레이어에 적용되지 않습니다.)
*   **일시 중단:** 잠시 원래 레시피로 빌드해야 할 때 (`--no-clean`과 함께 사용하여 소스는 보존).

## 5. `devtool finish`와의 차이점

*   **`devtool finish`**: 변경 사항(패치, 레시피 수정)을 원본 레이어에 **적용**하고 워크스페이스에서 제거합니다.
*   **`devtool reset`**: 변경 사항을 원본 레이어에 **적용하지 않고** 워크스페이스에서 제거합니다. 소스 디렉토리의 커밋되지 않은 변경 사항은 삭제될 수 있습니다.

---
*참고: `devtool reset`을 실행하면 워크스페이스의 `appends` 및 `recipes` 디렉토리에서 관련 파일이 삭제됩니다.*
