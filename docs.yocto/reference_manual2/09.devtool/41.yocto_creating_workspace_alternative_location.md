# Yocto Project 워크스페이스 레이어 대체 위치 생성 (Creating the Workspace Layer in an Alternative Location)

Yocto Project에서 `devtool`이 사용하는 워크스페이스 레이어를 기본 위치(`build/workspace`)가 아닌 다른 경로에 생성하는 방법에 대해 설명합니다.

## 1. 개요

기본적으로 `devtool`은 빌드 디렉토리 내의 `workspace` 서브 디렉토리를 워크스페이스 레이어로 사용합니다. 하지만 소스 코드를 빌드 디렉토리 외부에서 관리하고 싶거나, 여러 빌드 프로젝트 간에 워크스페이스를 공유하고 싶은 경우 대체 위치를 지정할 수 있습니다.

## 2. 명령어 사용법

`devtool create-workspace` 명령어를 사용하여 새로운 위치에 워크스페이스 레이어를 생성할 수 있습니다.

```bash
devtool create-workspace <layerpath>
```

*   **`<layerpath>`**: 워크스페이스 레이어를 생성할 절대 경로 또는 상대 경로입니다.

## 3. 사용 예시

워크스페이스를 홈 디렉토리의 `my-shared-workspace`에 생성하고 싶다면 다음과 같이 실행합니다.

```bash
devtool create-workspace ~/my-shared-workspace
```

## 4. 작동 방식

이 명령을 실행하면 다음과 같은 작업이 수행됩니다.

1.  **디렉토리 생성:** 지정된 경로에 레이어 디렉토리 구조(`conf`, `recipes` 등)를 생성합니다.
2.  **설정 파일 생성:** `conf/layer.conf` 파일을 생성하여 레이어를 정의하고, `bbappend` 파일들이 적용되도록 설정합니다.
3.  **레이어 추가:** 현재 빌드 환경의 `conf/bblayers.conf` 파일에 새로 생성된 워크스페이스 레이어의 경로를 추가합니다.

## 5. 주의 사항

*   이미 워크스페이스 레이어가 설정된 상태에서 다른 위치를 지정하면 에러가 발생할 수 있습니다. 기존 워크스페이스가 있다면 먼저 정리하거나 `bblayers.conf`에서 제거해야 할 수 있습니다.

---
*참고: 이 기능은 소스 코드를 빌드 아티팩트와 분리하여 관리하고자 할 때 유용합니다.*
