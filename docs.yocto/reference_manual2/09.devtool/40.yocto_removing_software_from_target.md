# Yocto Project 타겟 장비에서 소프트웨어 제거 (Removing Your Software from the Target Machine)

Yocto Project에서 `devtool undeploy-target` 명령어를 사용하여 타겟 장비에 배포했던 소프트웨어를 제거하는 방법에 대해 설명합니다.

## 1. 개요

`devtool deploy-target`을 사용하여 타겟 장비에 배포한 파일들은 테스트가 끝난 후 정리해야 할 수 있습니다. `devtool undeploy-target` 명령은 이전에 배포된 파일들을 타겟 장비에서 삭제하고, 시스템을 배포 이전 상태(해당 파일들이 없는 상태)로 되돌리는 역할을 합니다.

## 2. 기본 사용법

```bash
devtool undeploy-target [options] <recipe> <target>
```

*   **`<recipe>`**: 제거할 소프트웨어의 레시피 이름입니다.
*   **`<target>`**: 타겟 장비의 주소입니다 (`user@ip` 형식, 예: `root@192.168.1.10`).

## 3. 주요 옵션

### 3.1 상태 확인 (`--show-status`, `-s`)
실제로 삭제하기 전에, 어떤 파일들이 제거될 대상인지 목록을 확인합니다.

```bash
devtool undeploy-target -s my-recipe root@192.168.1.10
```

### 3.2 드라이 런 (`--dry-run`, `-n`)
실제로 삭제 명령을 실행하지 않고, 어떤 작업이 수행될지 시뮬레이션만 수행합니다.

```bash
devtool undeploy-target -n my-recipe root@192.168.1.10
```

### 3.3 에러 무시 (`--no-check-space`, `-c`)
디스크 공간 확인 등을 건너뛰거나 사소한 에러를 무시하고 진행할 때 사용할 수 있습니다 (버전에 따라 옵션이 다를 수 있음).

## 4. 작동 방식

1.  `devtool`은 해당 레시피에 대해 이전에 `deploy-target`으로 배포된 파일 목록을 조회합니다.
2.  SSH를 통해 타겟 장비에 접속합니다.
3.  해당 파일들을 타겟 파일시스템에서 삭제합니다.

---
*참고: 이 명령은 `deploy-target`으로 복사된 파일만 제거합니다. 만약 배포 과정에서 기존 시스템 파일을 덮어썼다면, `undeploy-target`이 원본 파일을 복구해주지는 않습니다.*
