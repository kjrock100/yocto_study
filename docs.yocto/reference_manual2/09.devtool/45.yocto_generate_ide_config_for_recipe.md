# Yocto Project 레시피 IDE 구성 생성 (Generate an IDE Configuration for a Recipe)

Yocto Project에서 `devtool ide-sdk` 명령어를 사용하여 레시피 개발을 위한 IDE(통합 개발 환경) 설정을 자동으로 생성하는 방법에 대해 설명합니다.

## 1. 개요

`devtool ide-sdk` 명령은 워크스페이스에 있는 레시피를 위해 SDK(Software Development Kit)를 준비하고, 이를 사용하는 IDE 구성 파일을 생성합니다. 이를 통해 개발자는 크로스 컴파일 환경 설정의 복잡함 없이 VSCode 등의 IDE에서 코드 작성, 컴파일, 디버깅을 수행할 수 있습니다.

## 2. 기본 사용법

```bash
devtool ide-sdk [options] <recipe> <ide>
```

*   **`<recipe>`**: IDE 구성을 생성할 레시피의 이름입니다. (먼저 `devtool modify` 상태여야 합니다.)
*   **`<ide>`**: 대상 IDE입니다. 현재 주로 `code` (Visual Studio Code)가 지원됩니다. `cmake`나 `none`을 지정할 수도 있습니다.

## 3. 사용 예시 (VSCode)

`my-app`이라는 레시피를 수정하고 VSCode 환경을 설정하는 과정입니다.

1.  **레시피 수정 모드 진입:**
    ```bash
    devtool modify my-app
    ```

2.  **IDE 구성 생성:**
    ```bash
    devtool ide-sdk my-app code
    ```
    이 명령은 필요한 SDK 구성 요소를 빌드하거나 캐시에서 가져오고, 소스 디렉토리에 `.vscode` 폴더와 설정 파일(`c_cpp_properties.json`, `launch.json`, `tasks.json` 등)을 생성합니다.

3.  **VSCode 실행:**
    ```bash
    code build/workspace/sources/my-app
    ```
    이제 VSCode에서 인텔리센스(IntelliSense)가 작동하며, 크로스 컴파일러 경로와 헤더 파일 경로가 올바르게 인식됩니다.

## 4. 주요 기능

*   **IntelliSense 설정:** 크로스 컴파일러의 시스템 헤더 경로와 레시피의 의존성 헤더 경로를 자동으로 설정합니다.
*   **빌드 작업:** IDE 내에서 `devtool build`를 실행하는 태스크를 구성할 수 있습니다.
*   **디버깅 (GDB):** `gdbserver`를 사용한 원격 디버깅 구성을 생성하여 타겟 장비 또는 QEMU에서의 디버깅을 돕습니다.

## 5. 지원되는 IDE 모드

*   **code**: Visual Studio Code용 설정을 생성합니다.
*   **cmake**: CMake 프로젝트를 위한 `CMakePresets.json` 등을 생성합니다 (레시피가 CMake를 사용하는 경우).
*   **none**: SDK만 준비하고 IDE 설정 파일은 생성하지 않습니다.

---
*참고: 이 기능을 사용하려면 `meta-ide-support` 등의 타겟이 빌드 가능해야 하며, 이미지에 `gdbserver`가 포함되어 있어야 디버깅이 원활할 수 있습니다.*
