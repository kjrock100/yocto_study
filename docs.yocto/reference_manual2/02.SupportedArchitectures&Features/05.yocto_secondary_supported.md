# Yocto Project 보조 지원 (Secondary Supported) 아키텍처

Yocto Project에서 **보조 지원(Secondary Supported)** 아키텍처는 프로젝트 내에서 지원되지만, 기본 지원(Primary Supported) 아키텍처만큼 엄격하거나 빈번하게 테스트되지 않는 하드웨어 아키텍처를 의미합니다.

## 1. 정의

보조 지원 아키텍처는 Yocto Project의 공식 릴리스 엔지니어링 팀이 직접적인 QA(품질 보증) 책임을 지지 않을 수 있는 아키텍처입니다. 대신, 이러한 아키텍처에 대한 지원은 주로 특정 하드웨어 벤더나 오픈 소스 커뮤니티의 기여에 의존합니다.

## 2. 주요 특징

*   **유지 관리 주체:** 주로 해당 하드웨어를 제조하는 벤더(Vendor)나 커뮤니티 멤버가 BSP(Board Support Package) 레이어를 유지 관리합니다.
*   **테스트 빈도:** 릴리스 주기마다 모든 테스트 케이스가 실행되지 않을 수 있으며, '기본 지원' 아키텍처에 비해 테스트 범위가 좁을 수 있습니다.
*   **릴리스 차단 여부:** 보조 지원 아키텍처에서의 빌드 실패나 버그가 Yocto Project의 전체 릴리스 일정을 지연시키거나 차단하지 않을 수 있습니다.

## 3. 사용 방법

보조 지원 아키텍처를 사용하기 위해서는 일반적으로 해당 아키텍처를 지원하는 별도의 **BSP 레이어**를 빌드 환경에 추가해야 합니다.

예를 들어:
*   특정 SoC 제조사의 `meta-vendor` 레이어 추가
*   `local.conf`에서 해당 레이어가 제공하는 `MACHINE` 설정

## 4. 요약

보조 지원 아키텍처는 Yocto Project 생태계의 중요한 부분이지만, 사용자는 기본 지원 아키텍처에 비해 발생할 수 있는 이슈에 대해 커뮤니티나 벤더의 지원을 직접 찾아야 할 가능성이 높습니다.
