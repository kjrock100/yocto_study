# Yocto Project 지원 아키텍처 및 기능

Yocto Project는 다양한 하드웨어 아키텍처를 지원하며, 이를 통해 개발자는 다양한 플랫폼에서 임베디드 리눅스 시스템을 구축할 수 있습니다. 지원 수준은 크게 기본 지원(Primary), 보조 지원(Secondary), 테스트되지 않음(Untested)으로 분류됩니다.

## 1. 기본 지원 (Primary Supported)

기본 지원 아키텍처는 Yocto Project 릴리스 엔지니어링 팀에 의해 지속적으로 빌드되고 테스트되는 아키텍처입니다. 이들은 주로 QEMU 에뮬레이터를 통해 검증되며, 참조 하드웨어에서도 테스트됩니다.

주요 아키텍처는 다음과 같습니다:

*   **ARM:** ARMv7 (32-bit), ARMv8 (64-bit / AArch64)
*   **x86:** x86 (32-bit), x86-64 (64-bit)
*   **MIPS:** MIPS32, MIPS64
*   **PowerPC:** PPC32, PPC64

이 아키텍처들은 `meta-yocto-bsp` 레이어와 `openembedded-core` 내의 QEMU 머신 정의(`qemux86`, `qemuarm` 등)를 통해 즉시 사용할 수 있습니다.

## 2. 보조 지원 (Secondary Supported)

보조 지원 아키텍처는 Yocto Project 내에서 지원되지만, 기본 아키텍처만큼 빈번하게 또는 엄격하게 테스트되지 않을 수 있는 아키텍처입니다. 종종 특정 하드웨어 벤더나 커뮤니티가 유지 관리하는 BSP 레이어를 통해 지원이 제공됩니다.

## 3. 테스트되지 않음 (Untested)

테스트되지 않은 아키텍처는 OpenEmbedded 빌드 시스템이 이론적으로 지원할 수 있으나(GCC 및 Linux 커널 지원 포함), Yocto Project의 공식 QA 주기 동안 검증되지 않는 아키텍처입니다. 이러한 아키텍처를 사용하려면 사용자가 직접 설정을 구성하거나 별도의 커뮤니티 레이어를 찾아야 할 수 있습니다.
