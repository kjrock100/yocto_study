# 변수 용어집 (Variables Glossary)

BitBake는 빌드 프로세스의 모든 측면을 제어하기 위해 수많은 변수를 사용합니다. 이 용어집은 가장 자주 사용되는 표준 변수들에 대한 참조를 제공합니다.

## 7.1 주요 변수 (Key Variables)

### 7.1.1 설정 및 경로 관련 (Configuration and Paths)

*   **`DL_DIR`**: 다운로드된 소스 파일(tarballs)이 저장되는 중앙 디렉토리입니다.
*   **`SSTATE_DIR`**: 공유 상태(Shared State) 캐시 파일이 저장되는 디렉토리입니다.
*   **`TMPDIR`**: 빌드 중 생성되는 모든 임시 파일, 작업 디렉토리, sysroot 등이 위치하는 최상위 디렉토리입니다.
*   **`WORKDIR`**: 특정 레시피의 빌드 작업이 수행되는 디렉토리입니다.
*   **`S`**: 소스 코드가 압축 해제된 위치입니다. (Source)
*   **`B`**: 실제 컴파일이 수행되는 디렉토리입니다. (Build). 기본값은 `S`와 같지만, 별도의 빌드 디렉토리를 사용하는 경우 다를 수 있습니다.
*   **`D`**: `do_install` 태스크가 파일을 설치하는 목적지 디렉토리입니다. (Destination)

### 7.1.2 레시피 정보 (Recipe Information)

*   **`PN`**: 패키지 이름 (Package Name).
*   **`PV`**: 패키지 버전 (Package Version).
*   **`PR`**: 패키지 리비전 (Package Revision). 주로 `r0`, `r1` 등으로 표시됩니다.
*   **`BP`**: `BPN`과 `PV`를 하이픈으로 연결한 값입니다. (예: `busybox-1.32.0`)
*   **`FILE`**: 현재 파싱 중인 파일의 절대 경로입니다.
*   **`FILESPATH`**: `SRC_URI`에 지정된 로컬 파일을 찾을 때 검색하는 경로 목록입니다.

### 7.1.3 빌드 제어 및 의존성 (Build Control and Dependencies)

*   **`SRC_URI`**: 소스 파일의 위치 목록입니다.
*   **`SRCREV`**: Git 등의 SCM에서 가져올 특정 리비전(커밋 해시)입니다.
*   **`DEPENDS`**: 빌드 시점 의존성 목록입니다.
*   **`RDEPENDS`**: 런타임 의존성 목록입니다. 패키지별로 설정해야 합니다 (예: `RDEPENDS:${PN}`).
*   **`PROVIDES`**: 레시피가 제공하는 기능이나 가상 패키지의 이름 목록입니다.
*   **`PACKAGES`**: 레시피가 생성할 패키지들의 목록입니다. 기본값은 `${PN}-src ${PN}-dbg ${PN}-staticdev ${PN}-dev ${PN}-doc ${PN}-locale ${PN}` 등입니다.
*   **`FILES`**: 각 패키지에 포함될 파일들의 목록입니다. (예: `FILES:${PN}`)

### 7.1.4 컴파일 및 설치 옵션 (Compile and Install Options)

*   **`EXTRA_OECONF`**: `do_configure` 단계에서 `./configure` 스크립트에 전달할 추가 인자입니다.
*   **`EXTRA_OEMAKE`**: `do_compile` 단계에서 `make` 명령에 전달할 추가 인자입니다.
*   **`CFLAGS`, `CXXFLAGS`, `LDFLAGS`**: C/C++ 컴파일러 및 링커 플래그입니다.

이 외에도 수많은 변수들이 존재하며, Yocto Project 공식 문서의 Variables Glossary에서 전체 목록을 확인할 수 있습니다.
