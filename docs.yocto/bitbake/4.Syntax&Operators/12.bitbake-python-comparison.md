# BitBake 스타일 파이썬 함수 vs 파이썬 함수 (BitBake-Style Python Functions Versus Python Functions)

BitBake에서는 두 가지 방식의 파이썬 함수 정의를 지원합니다. 각각의 특징과 차이점을 이해하고 적절한 상황에 사용하는 것이 중요합니다.

## 4.5.4 비교 (Comparison)

| 특징 | BitBake 스타일 파이썬 함수 | 표준 파이썬 함수 |
| :--- | :--- | :--- |
| **정의 문법** | `python funcname () { ... }` | `def funcname(args): ...` |
| **`d` (Datastore) 접근** | 전역 변수처럼 자동 접근 가능 | 인자로 명시적으로 전달받아야 함 (`args`) |
| **주요 용도** | 태스크(`do_task`), 이벤트 핸들러 | 유틸리티 함수, 인라인 변수 확장(`${@...}`) |
| **호출 방식** | BitBake 엔진에 의해 실행 (태스크/핸들러) | 다른 파이썬 코드나 변수 확장 내에서 호출 |
| **오버라이드 지원** | `:append`, `:prepend` 등 지원 | 지원하지 않음 (일반 파이썬 모듈처럼 동작) |

### 4.5.4.1 차이점 상세

1.  **실행 환경 (Execution Environment)**
    *   **BitBake 스타일**: BitBake가 파싱하고 실행을 관리하는 특수한 환경에서 동작합니다. `bb.note()` 등을 바로 사용할 수 있고, `d` 객체가 자동으로 주입됩니다.
    *   **표준 파이썬**: 일반적인 파이썬 모듈 스코프 내에서 정의됩니다.

2.  **데이터 접근 (Accessing Data)**
    *   **BitBake 스타일**: `d.getVar('VAR')`를 바로 호출할 수 있습니다.
    *   **표준 파이썬**: 함수 정의 시 `def my_func(d):`와 같이 인자를 정의하고, 호출할 때 `d`를 넘겨줘야 합니다.

    ```python
    # BitBake Style
    python do_print_date() {
        # d가 자동으로 사용 가능
        print(d.getVar("DATE"))
    }

    # Standard Python
    def get_date(d):
        # d를 인자로 받아야 함
        return d.getVar("DATE")
    
    # 호출 시 d를 전달
    MY_DATE = "${@get_date(d)}"
    ```

3.  **확장성 (Extensibility)**
    *   **BitBake 스타일**: 다른 레이어에서 `.bbappend` 파일을 통해 `do_task:append()`와 같이 기능을 쉽게 확장할 수 있습니다.
    *   **표준 파이썬**: 일반적인 파이썬 함수이므로 BitBake의 오버라이드 문법이 적용되지 않습니다.
