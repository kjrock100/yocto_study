# 태스크 (Tasks)

BitBake에서 태스크는 실행의 단위입니다. 레시피 내의 함수를 태스크로 승격시켜 BitBake가 의존성 그래프에 포함시키고 실행할 수 있게 만듭니다.

## 4.6.1 함수를 태스크로 승격시키기 (Promoting a Function to a Task)

레시피나 클래스 파일에 정의된 함수(Shell 또는 Python)를 태스크로 만들기 위해서는 `addtask` 명령어를 사용해야 합니다. 이 명령어를 통해 해당 함수를 빌드 실행 대기열(RunQueue)에 추가하고 다른 태스크와의 실행 순서를 정의할 수 있습니다.

### 사용법 (Usage)

```bitbake
addtask taskname [after existing_task] [before existing_task]
```

*   `taskname`: 태스크로 승격시킬 함수의 이름입니다.
*   `after`: 이 태스크가 실행되기 전에 완료되어야 하는 태스크를 지정합니다. (의존성 설정)
*   `before`: 이 태스크가 완료된 후에 실행되어야 하는 태스크를 지정합니다.

### 예제 (Example)

```bitbake
do_print_date() {
    date
}

addtask do_print_date after do_fetch before do_build
```

위 예제에서 `do_print_date` 함수는 `do_fetch` 태스크가 완료된 후, `do_build` 태스크가 시작되기 전에 실행되는 태스크로 등록됩니다.

### 참고 사항

*   태스크 이름은 관례적으로 `do_` 접두사를 사용합니다.
*   `addtask`를 사용하지 않은 함수는 단순히 내부적으로 호출 가능한 함수일 뿐이며, 사용자가 직접 `bitbake -c` 명령어로 호출하거나 빌드 흐름에 자동으로 포함되지 않습니다.
