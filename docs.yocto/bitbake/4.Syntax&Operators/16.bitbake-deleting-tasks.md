# 태스크 삭제 (Deleting a Task)

특정 레시피에서 불필요한 태스크를 제거하거나, 상속받은 클래스의 기본 태스크를 비활성화해야 할 때가 있습니다. BitBake는 이를 위해 `deltask` 명령어를 제공합니다.

## 4.6.2 태스크 삭제 (Deleting a Task)

`deltask` 지시어를 사용하면 태스크를 빌드 프로세스에서 완전히 제거할 수 있습니다. 이는 해당 태스크가 실행되지 않을 뿐만 아니라, 의존성 그래프에서도 사라진다는 것을 의미합니다.

### 사용법 (Usage)

```bitbake
deltask taskname
```

*   `taskname`: 삭제하고자 하는 태스크의 이름입니다.

### 예제 (Example)

예를 들어, 소스 코드가 없는 바이너리 전용 레시피의 경우 `do_configure`나 `do_compile` 단계가 필요 없을 수 있습니다.

```bitbake
deltask do_configure
deltask do_compile
```

### 주의사항 및 대안 (Notes and Alternatives)

`deltask`는 태스크를 의존성 체인에서 완전히 끊어버립니다. 만약 태스크의 실행만 건너뛰고 의존성 관계(순서)는 유지해야 한다면, `deltask` 대신 `noexec` 플래그를 사용하는 것이 안전합니다.

```bitbake
# 태스크는 존재하지만 실행되지는 않음 (의존성 유지)
do_configure[noexec] = "1"
```

또는 함수 본문을 비워서 아무 작업도 하지 않게 만들 수도 있습니다.

```bitbake
do_configure() {
    :
}
```
