# 다중 소스 저장소 (Multiple Source Control Repositories)

하나의 레시피에서 메인 소스 코드 외에 다른 저장소의 코드가 필요한 경우가 있습니다. 예를 들어, 별도의 서브모듈을 직접 관리하거나, 펌웨어 바이너리 등을 다른 저장소에서 가져와야 할 때입니다. BitBake는 `SRC_URI`에 여러 저장소를 지정하여 이를 지원합니다.

## 6.5.1 사용법 (Usage)

`SRC_URI` 변수에 여러 개의 `git://` 항목을 나열합니다. 이때 각 저장소를 구분하고 체크아웃 위치를 지정하기 위해 파라미터를 사용해야 합니다.

### 주요 파라미터

*   **`name`**: 각 URI 항목에 고유한 이름을 부여합니다. 이는 나중에 `SRCREV`를 지정할 때 사용됩니다.
*   **`destsuffix`**: 해당 저장소의 코드가 압축 해제(체크아웃)될 디렉토리 경로를 지정합니다. 경로는 `WORKDIR`에 대한 상대 경로입니다.

## 6.5.2 예제 (Example)

다음은 두 개의 Git 저장소(메인 애플리케이션과 유틸리티 라이브러리)를 사용하는 예제입니다.

```bitbake
SRC_URI = "git://example.com/main-app.git;protocol=https;branch=master;name=main \
           git://example.com/utils-lib.git;protocol=https;branch=stable;name=utils;destsuffix=git/utils"

# 각 저장소에 대한 리비전 지정
SRCREV_main = "abcdef1234567890..."
SRCREV_utils = "1234567890abcdef..."

# 메인 소스 디렉토리 설정 (필요한 경우)
S = "${WORKDIR}/git"
```

### 설명

1.  **`name=main`**: 첫 번째 저장소의 이름을 'main'으로 지정했습니다. 이에 대한 커밋 해시는 `SRCREV_main` 변수로 설정합니다.
2.  **`name=utils`**: 두 번째 저장소의 이름을 'utils'로 지정했습니다. 커밋 해시는 `SRCREV_utils`로 설정합니다.
3.  **`destsuffix=git/utils`**: 두 번째 저장소는 `WORKDIR/git/utils` 디렉토리에 체크아웃됩니다. 첫 번째 저장소는 `destsuffix`가 없으므로 기본 위치(주로 `git`)에 체크아웃됩니다.

이 방식을 사용하면 복잡한 소스 구성을 가진 소프트웨어도 하나의 레시피 내에서 유연하게 관리할 수 있습니다.
