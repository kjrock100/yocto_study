# 로깅 유틸리티 (Logging Utilities)

BitBake는 빌드 프로세스 중 발생하는 정보를 사용자에게 알리고 로그 파일에 기록하기 위해 `bb` 모듈 내에 다양한 로깅 함수를 제공합니다. 이 함수들은 파이썬 코드(태스크, 핸들러, 클래스 등) 내에서 사용할 수 있습니다.

## 8.1 주요 로깅 함수 (Key Logging Functions)

### `bb.plain(msg)`

추가적인 포맷팅이나 로그 레벨 표시 없이 일반 텍스트 메시지를 출력합니다. 로그 파일과 콘솔에 그대로 기록됩니다.

### `bb.note(msg)`

정보성 메시지(Note)를 출력합니다. 실행 흐름을 추적하거나 일반적인 상태 정보를 알릴 때 사용합니다.

### `bb.warn(msg)`

경고 메시지(Warning)를 출력합니다. 잠재적인 문제가 있지만 빌드를 중단할 정도는 아닐 때 사용합니다.

### `bb.error(msg)`

에러 메시지(Error)를 출력합니다. 심각한 문제가 발생했음을 알립니다. 이 함수가 호출된다고 해서 즉시 실행이 중단되지는 않지만, 해당 태스크는 실패(Failed) 상태로 종료될 가능성이 높습니다.

### `bb.fatal(msg)`

치명적인 에러(Fatal Error)를 출력하고 **즉시 실행을 중단**합니다. 복구할 수 없는 오류가 발생했을 때 사용합니다. 내부적으로 예외를 발생시킵니다.

### `bb.debug(level, msg)`

디버그 레벨에 따라 메시지를 출력합니다.

* `level`: 1, 2, 3 등의 정수값.
* 사용자가 `bitbake -D` (레벨 1), `-DD` (레벨 2) 등의 옵션을 사용했을 때만 출력됩니다.

```python
bb.debug(1, "This appears with -D")
bb.debug(2, "This appears with -DD")
```

## 8.1.1 중복 방지 함수 (Once Functions)

동일한 메시지가 반복적으로 출력되는 것을 방지하기 위한 함수들입니다. 루프 내에서나 여러 번 호출될 수 있는 함수에서 유용합니다.

### `bb.warnonce(msg)`

`bb.warn(msg)`과 유사하지만, 동일한 내용의 메시지는 빌드 세션 동안 **단 한 번만** 출력됩니다.

### `bb.erroronce(msg)`

`bb.error(msg)`와 유사하지만, 동일한 내용의 메시지는 **단 한 번만** 출력됩니다.

## 8.1.2 기타 함수

### `bb.verbnote(msg)`

`bb.note(msg)`와 유사하지만, 사용자가 상세 모드(verbose)를 활성화했을 때만 출력됩니다.

## 8.1.3 사용 예시 (Example)

```python
python do_check_config() {
    if not d.getVar("REQUIRED_VAR"):
        bb.fatal("REQUIRED_VAR is not set!")
    
    if d.getVar("DEPRECATED_VAR"):
        bb.warnonce("DEPRECATED_VAR is deprecated, please use NEW_VAR instead.")
}
```
