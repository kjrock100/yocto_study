# Yocto Project 개발 환경 (The Yocto Project Development Environment)

Yocto Project 개발 환경은 단순히 컴파일러를 설치하는 것을 넘어, 오픈 소스 커뮤니티와의 협업, Git을 이용한 소스 관리, 그리고 라이선스 준수까지 포괄하는 개념입니다.

## 1. 오픈 소스 철학 (Open Source Philosophy)

Yocto Project는 강력한 오픈 소스 철학을 바탕으로 운영됩니다.

*   **협업 (Collaboration):** 전 세계의 개발자들이 메일링 리스트와 IRC 등을 통해 소통하며 프로젝트를 발전시킵니다.
*   **업스트림 퍼스트 (Upstream First):** 버그를 수정하거나 기능을 추가할 때, Yocto Project 내부에서만 해결하는 것이 아니라 해당 소프트웨어의 원본 프로젝트(Upstream)에 먼저 기여하는 것을 권장합니다. 이는 유지 보수 부담을 줄이고 오픈 소스 생태계 전체에 기여하는 방식입니다.

## 2. 개발 호스트 (The Development Host)

Yocto Project 빌드 시스템(BitBake)을 실행하기 위한 로컬 컴퓨터 환경입니다.

*   **운영체제:** 리눅스 기반 시스템이 필요합니다. 공식적으로 Ubuntu, Fedora, Debian, openSUSE 등이 지원됩니다. (Windows나 macOS 사용자는 가상 머신이나 WSL2, Docker 등을 사용해야 합니다.)
*   **필수 패키지:** 빌드 도구(`gcc`, `make`), 버전 관리(`git`), 스크립트 언어(`python3`), 그리고 기타 유틸리티(`tar`, `diffstat` 등)를 설치해야 합니다.
*   **시스템 자원:** Yocto 빌드는 많은 CPU 코어와 메모리, 그리고 수십 GB 이상의 디스크 공간을 필요로 합니다.

## 3. Yocto Project 소스 저장소 (Source Repositories)

Yocto Project의 모든 메타데이터와 소스 코드는 Git 저장소(Repository) 형태로 관리됩니다.

*   **Poky:** 빌드 시스템과 핵심 레이어를 포함하는 메인 저장소입니다.
*   **레이어 저장소:** `meta-raspberrypi`, `meta-qt5` 등 다양한 하드웨어 및 소프트웨어 레이어들이 별도의 Git 저장소로 존재합니다. OpenEmbedded Layer Index에서 검색할 수 있습니다.

## 4. Git 워크플로우 (Git Workflows)

Yocto Project는 Git을 적극적으로 활용합니다.

*   **브랜치 (Branches):** Yocto Project는 주기적으로 새로운 릴리스(예: `kirkstone`, `scarthgap`)를 발표하며, 각 릴리스는 고유한 브랜치 이름을 가집니다. 프로젝트의 안정성을 위해 호환되는 브랜치를 맞춰서 사용해야 합니다.
*   **태그 (Tags):** 특정 릴리스 시점이나 마일스톤을 표시합니다.

## 5. 라이선스 (Licensing)

상용 제품을 개발할 때 오픈 소스 라이선스 관리는 필수적입니다.

*   **라이선스 추적:** 빌드 시스템은 생성된 이미지에 포함된 모든 패키지의 라이선스 정보를 수집합니다.
*   **제한 설정:** `local.conf`의 `INCOMPATIBLE_LICENSE` 변수를 사용하여 특정 라이선스(예: `GPL-3.0`)를 가진 패키지가 빌드에 포함되지 않도록 막을 수 있습니다.
*   **매니페스트 생성:** 배포 시 필요한 라이선스 고지문을 자동으로 생성할 수 있습니다.

---
*참고: 성공적인 Yocto Project 개발을 위해서는 이러한 환경적 요소와 도구들에 대한 이해가 선행되어야 합니다.*
