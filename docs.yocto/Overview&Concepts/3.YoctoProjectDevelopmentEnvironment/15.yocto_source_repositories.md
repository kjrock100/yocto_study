# Yocto Project 소스 저장소 (Source Repositories)

Yocto Project는 소스 코드 중심의 개발 환경을 제공하며, 모든 구성 요소는 **Git 저장소(Source Repositories)**를 통해 관리됩니다.

## 1. 개요

Yocto Project의 생태계는 단일 저장소가 아닌, 수많은 저장소들의 집합으로 이루어져 있습니다. 핵심 빌드 시스템, 메타데이터, 문서, 그리고 다양한 레이어들이 각각 독립적인 Git 저장소로 존재합니다.

## 2. 주요 저장소 (Key Repositories)

이 저장소들은 주로 `git.yoctoproject.org`에서 호스팅됩니다.

### 2.1 Poky (`poky`)
*   **역할:** Yocto Project의 **레퍼런스 배포판**이자 통합 빌드 시스템입니다.
*   **내용:** 사용자가 빌드를 시작하기 위해 필요한 모든 것(BitBake, OE-Core, 기본 레이어)을 포함하고 있습니다.
*   **사용:** 대부분의 개발자는 이 저장소를 클론(`git clone`)하여 프로젝트를 시작합니다.
*   **URL:** `git://git.yoctoproject.org/poky`

### 2.2 OpenEmbedded-Core (`openembedded-core`)
*   **역할:** Yocto Project와 OpenEmbedded 프로젝트가 공유하는 **핵심 메타데이터**입니다.
*   **내용:** `meta/` 디렉토리에 해당하는 내용으로, 기본 레시피와 클래스들을 담고 있습니다.
*   **관계:** Poky 저장소는 이 OE-Core의 복사본을 포함하고 있습니다.

### 2.3 BitBake (`bitbake`)
*   **역할:** 작업을 스케줄링하고 실행하는 **빌드 엔진**입니다.
*   **내용:** 파이썬으로 작성된 빌드 도구 소스 코드입니다.
*   **관계:** Poky 저장소는 특정 버전의 BitBake를 포함하고 있습니다.

### 2.4 Meta-Yocto (`meta-yocto`)
*   **역할:** Poky 배포판을 위한 정책과 레퍼런스 하드웨어 설정을 담고 있습니다.
*   **내용:** `meta-poky`(배포판 설정)와 `meta-yocto-bsp`(보드 지원 패키지) 레이어를 포함합니다.

## 3. 레이어 저장소 (Layer Repositories)

핵심 저장소 외에도, 특정 하드웨어나 소프트웨어를 지원하기 위한 수많은 레이어 저장소가 존재합니다.

*   **OpenEmbedded Layer Index:** 전 세계의 다양한 레이어를 검색할 수 있는 인덱스 사이트입니다. (http://layers.openembedded.org/)
*   **예시:**
    *   `meta-intel`, `meta-ti`, `meta-raspberrypi` (하드웨어 지원)
    *   `meta-qt5`, `meta-browser`, `meta-security` (소프트웨어 기능 추가)

## 4. 웹 인터페이스 (Source Repositories Web Interface)

Git 명령어를 사용하지 않고도 웹 브라우저를 통해 저장소를 탐색할 수 있습니다.

*   **주소:** http://git.yoctoproject.org/
*   **기능:** 프로젝트별 저장소 목록, 커밋 로그, 브랜치/태그 정보, 파일 내용 조회 등을 제공합니다.

## 5. 브랜치 관리 (Branching)

Yocto Project는 6개월마다 새로운 릴리스를 발표하며, 각 릴리스는 별도의 브랜치(예: `scarthgap`, `kirkstone`)로 관리됩니다. 프로젝트 구성 시 **모든 레이어의 브랜치를 일치**시키는 것이 중요합니다.

---
*참고: 소스 저장소를 이해하는 것은 Yocto Project의 구성 요소를 파악하고 필요한 코드를 찾는 첫걸음입니다.*
