# Yocto Project 개발 호스트 (The Development Host)

Yocto Project를 사용하여 이미지를 빌드하기 위해서는 **개발 호스트(Development Host)**라고 불리는 로컬 컴퓨터 환경을 적절히 구성해야 합니다.

## 1. 개요

Yocto Project의 빌드 시스템인 BitBake는 리눅스 환경에서 실행되도록 설계되었습니다. 따라서 개발 호스트는 리눅스 운영체제를 실행해야 하며, 빌드에 필요한 필수 소프트웨어 패키지들이 설치되어 있어야 합니다.

## 2. 지원하는 리눅스 배포판 (Supported Linux Distributions)

Yocto Project는 다양한 리눅스 배포판을 공식적으로 지원합니다. 안정적인 빌드를 위해 **LTS (Long Term Support)** 버전을 사용하는 것이 권장됩니다.

*   **Ubuntu**
*   **Fedora**
*   **Debian**
*   **openSUSE**
*   **CentOS / AlmaLinux / Rocky Linux**

## 3. 필수 패키지 설치 (Required Packages)

빌드 시스템이 정상적으로 작동하려면 컴파일러, 버전 관리 도구, 스크립트 언어 등 다양한 호스트 도구가 필요합니다.

### Ubuntu/Debian 예시
```bash
sudo apt install gawk wget git diffstat unzip texinfo gcc build-essential \
chrpath socat cpio python3 python3-pip python3-pexpect xz-utils \
debianutils iputils-ping python3-git python3-jinja2 libegl1-mesa \
libsdl1.2-dev pylint3 xterm Python3-subunit mesa-common-dev zstd \
liblz4-tool
```

*   **`build-essential`**: GCC, Make 등 기본 빌드 도구.
*   **`python3`**: BitBake는 Python으로 작성되었습니다.
*   **`git`**: 소스 코드 다운로드 및 버전 관리.
*   **`diffstat`, `chrpath`**: 패치 및 RPATH 관리 도구.

## 4. 시스템 요구 사항 (System Requirements)

Yocto Project 빌드(특히 처음부터 빌드하는 경우)는 시스템 자원을 매우 많이 소모합니다.

*   **CPU:** 코어 수가 많을수록 빌드 속도가 빨라집니다. (BitBake는 병렬 처리에 최적화되어 있습니다.)
*   **RAM:** 최소 8GB 이상이 필요하며, 16GB 이상을 권장합니다. 메모리가 부족하면 링커(Linker) 오류가 발생할 수 있습니다.
*   **디스크 공간:** 최소 50GB의 여유 공간이 필요하며, 복잡한 이미지를 빌드할 경우 100GB 이상을 권장합니다. `tmp/` 디렉토리에 생성되는 임시 파일들의 용량이 매우 큽니다.

## 5. 비 리눅스 환경 (Non-Linux Environments)

Windows나 macOS를 사용하는 경우, 직접적으로 BitBake를 실행할 수 없으므로 다음과 같은 대안을 사용해야 합니다.

*   **WSL 2 (Windows Subsystem for Linux):** Windows 10/11에서 리눅스 커널을 직접 실행하여 Ubuntu 등을 설치해 사용할 수 있습니다. 성능이 우수하여 많이 사용됩니다.
*   **CROPS (Cross-Platform Support):** Docker 컨테이너를 사용하여 격리된 빌드 환경을 제공합니다. OS에 상관없이 일관된 환경을 구축할 수 있습니다.
*   **가상 머신 (Virtual Machine):** VMware나 VirtualBox를 사용하지만, 성능 오버헤드가 있어 빌드 속도가 느릴 수 있습니다.

---
*참고: 호스트 환경 설정이 잘못되면 빌드 초기에 "Sanity Check" 오류가 발생할 수 있습니다.*
