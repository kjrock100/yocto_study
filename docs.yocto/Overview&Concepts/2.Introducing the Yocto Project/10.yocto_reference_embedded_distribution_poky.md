# Yocto Project 레퍼런스 임베디드 배포판 (Reference Embedded Distribution - Poky)

Yocto Project에서 **Poky**는 단순한 빌드 시스템을 넘어, 실제로 동작하는 **레퍼런스 임베디드 배포판(Reference Embedded Distribution)**의 역할을 수행합니다.

## 1. 개념

Poky는 Yocto Project의 도구(BitBake)와 메타데이터(OpenEmbedded-Core)를 사용하여 구축할 수 있는 **예제 리눅스 배포판**입니다.

*   **구성:** 커널, 툴체인, 라이브러리, 그리고 Sato(모바일 스타일 UI)와 같은 애플리케이션을 포함합니다.
*   **의미:** "Yocto Project를 사용하면 이러한 형태의 배포판을 만들 수 있다"는 것을 보여주는 **동작 증명(Proof of Concept)**이자 **템플릿**입니다.

## 2. 배포판으로서의 역할

Poky는 다음과 같은 중요한 기능을 수행합니다.

### 2.1 정책(Policy) 정의의 예시
리눅스 배포판은 수많은 선택의 집합입니다. Poky는 `meta-poky` 레이어를 통해 이러한 **배포판 정책(Distro Policy)**을 어떻게 구성하는지 보여줍니다.
*   **초기화 시스템:** SysVinit 또는 systemd 선택.
*   **C 라이브러리:** glibc 또는 musl 선택.
*   **패키지 관리:** RPM, DEB, IPK 중 선택.
*   **툴체인 설정:** GCC 버전 및 컴파일 옵션.

이러한 설정은 주로 `conf/distro/poky.conf` 파일에 정의되어 있으며, 개발자는 이를 참고하여 자신만의 배포판 설정(`mydistro.conf`)을 작성할 수 있습니다.

### 2.2 테스트 및 검증 (Testing & Validation)
Yocto Project 개발 팀은 Poky를 사용하여 전체 시스템을 빌드하고 부팅 테스트를 수행합니다.
*   Poky가 정상적으로 빌드되고 동작한다면, 이를 구성하는 BitBake와 OE-Core가 안정적이라는 것을 의미합니다.
*   QEMU(에뮬레이터) 뿐만 아니라 실제 하드웨어 레퍼런스 보드에서도 테스트됩니다.

### 2.3 커스터마이징의 출발점 (Base for Customization)
대부분의 임베디드 프로젝트는 Poky를 기반으로 시작합니다.
*   개발자는 Poky를 그대로 제품에 탑재하기보다는, Poky의 구조를 복제하거나 상속받아 불필요한 기능을 제거하고 자사 제품에 필요한 기능을 추가하여 **커스텀 배포판**을 만듭니다.

## 3. Poky와 Yocto Project의 차이

혼동하기 쉽지만, 엄밀히 구분하면 다음과 같습니다.

*   **Yocto Project:** 임베디드 리눅스를 만들기 위한 오픈 소스 프로젝트, 도구, 커뮤니티 전체를 지칭합니다.
*   **Poky:** Yocto Project 내에서 제공되는 **구체적인 배포판의 구현체**입니다. (Fedora나 Ubuntu가 리눅스의 배포판인 것과 유사합니다.)

## 4. 요약

Poky는 Yocto Project가 제공하는 **"모범 답안"**입니다. 개발자는 이 모범 답안을 분석하여 배포판을 구성하는 방법을 배우고, 이를 수정하여 자신의 하드웨어와 요구사항에 맞는 최적의 리눅스 시스템을 구축하게 됩니다.
