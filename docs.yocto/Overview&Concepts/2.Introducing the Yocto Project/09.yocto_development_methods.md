# Yocto Project 개발 방법론 (Development Methods)

Yocto Project는 개발자가 임베디드 리눅스 시스템을 구축하고 애플리케이션을 개발할 수 있도록 다양한 방법론을 제공합니다. 크게 **시스템 개발**과 **애플리케이션 개발**로 나눌 수 있습니다.

## 1. 시스템 개발 (System Development)

시스템 개발은 보드 지원 패키지(BSP), 커널, 부트로더, 그리고 루트 파일시스템 이미지를 생성하는 작업을 포함합니다.

### 1.1 BitBake 및 레시피 기반 개발
가장 전통적이고 기본적인 방법입니다.
*   **작업:** 레시피(`.bb`)를 작성하거나 수정하고, `bitbake` 명령어를 사용하여 전체 이미지를 빌드합니다.
*   **장점:** 시스템의 모든 구성 요소를 완벽하게 제어할 수 있습니다.
*   **단점:** 빌드 시간이 오래 걸리며, 전체 빌드 시스템에 대한 이해가 필요합니다.

### 1.2 레이어 활용
기존 시스템을 수정할 때는 원본 파일을 직접 고치지 않고, 자신만의 레이어(`meta-mylayer`)를 만들어 `bbappend` 파일을 통해 설정을 오버라이드합니다. 이는 유지 보수성을 높여줍니다.

## 2. 애플리케이션 개발 (Application Development)

이미 구축된 시스템 위에서 동작하는 사용자 애플리케이션을 개발하는 방법입니다.

### 2.1 Standard SDK (Standard Software Development Kit)
크로스 컴파일 툴체인과 필요한 라이브러리, 헤더 파일만 포함된 독립적인 설치 파일입니다.
*   **워크플로우:**
    1.  `bitbake -c populate_sdk <image>` 명령으로 SDK 생성.
    2.  개발자 PC에 SDK 설치.
    3.  환경 설정 스크립트(`source environment-setup-...`) 실행.
    4.  Makefile이나 Autotools 등을 사용하여 애플리케이션 크로스 컴파일.
*   **특징:** Yocto 빌드 시스템(BitBake)과 독립적으로 작업할 수 있어 애플리케이션 개발자에게 익숙한 환경을 제공합니다.

### 2.2 Extensible SDK (eSDK) 및 devtool
Standard SDK의 기능에 더해, Yocto 빌드 시스템의 일부 기능(`devtool`)을 포함하여 더 강력한 개발 환경을 제공합니다.
*   **devtool:** 소스 코드를 가져와서 작업 공간(Workspace)에 풀고, 수정하고, 빌드하고, 타겟에 배포하는 과정을 자동화해주는 도구입니다.
*   **워크플로우:**
    1.  `devtool add <recipe> <source-url>`: 소스 가져오기 및 레시피 생성.
    2.  `devtool modify <recipe>`: 소스 코드 수정 모드 진입.
    3.  `devtool build <recipe>`: 빌드.
    4.  `devtool deploy-target <recipe> <target-ip>`: 타겟 장비에 배포 및 테스트.
    5.  `devtool finish <recipe> <layer>`: 변경 사항을 레시피에 반영(패치 생성 등).

## 3. 요약

| 방법 | 주 사용자 | 목적 | 도구 |
| :--- | :--- | :--- | :--- |
| **시스템 개발** | 시스템 통합자, BSP 엔지니어 | OS 이미지 생성, 하드웨어 지원 | BitBake, Recipes |
| **Standard SDK** | 애플리케이션 개발자 | 앱 크로스 컴파일 | GCC, Make, CMake |
| **eSDK / devtool** | 앱/시스템 개발자 | 소스 수정, 반복적인 테스트 | devtool |

*참고: 프로젝트의 단계와 목적에 따라 적절한 개발 방법을 선택하는 것이 중요합니다.*
