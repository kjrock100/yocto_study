# Yocto Project 레퍼런스 배포판 (Reference Distribution - Poky)

**Poky**는 Yocto Project의 **레퍼런스 배포판(Reference Distribution)**입니다.

## 1. 개요

Poky는 Yocto Project의 빌드 시스템과 메타데이터를 통합하여 제공하는 예제 시스템입니다. 이는 그 자체로 최종 제품에 사용되는 배포판이라기보다는, 사용자가 자신만의 커스텀 임베디드 리눅스 배포판을 구축하기 위한 **출발점(Starting Point)**이자 **테스트 베드** 역할을 합니다.

## 2. 구성 요소

Poky는 다음과 같은 핵심 요소들을 포함하고 있습니다.

*   **BitBake**: 작업을 스케줄링하고 실행하는 빌드 엔진.
*   **OpenEmbedded-Core (OE-Core)**: 기본 레시피, 클래스, 도구들을 포함하는 핵심 메타데이터 (`meta/` 디렉토리).
*   **meta-poky**: Poky 배포판의 정책(Policy)과 설정을 정의하는 레이어.
*   **meta-yocto-bsp**: QEMU(에뮬레이터) 및 일반적인 레퍼런스 하드웨어 보드를 지원하는 BSP 레이어.

## 3. 주요 역할

### 3.1 시스템 검증
Poky는 Yocto Project의 릴리스를 검증하는 데 사용됩니다. BitBake와 OE-Core의 변경 사항이 실제 빌드 환경에서 문제없이 작동하는지 확인하는 기준이 됩니다.

### 3.2 커스터마이징 베이스
개발자는 Poky를 그대로 사용하는 것이 아니라, 이를 복제하거나 수정하여 자신의 프로젝트에 맞는 배포판을 만듭니다. `local.conf`를 수정하거나 새로운 레이어를 추가하는 방식으로 확장합니다.

### 3.3 안정적인 기반 제공
Poky는 엄격한 테스트 과정을 거쳐 릴리스되므로, 개발자에게 안정적이고 신뢰할 수 있는 빌드 환경을 제공합니다.

## 4. Poky와 Yocto Project의 관계

*   **Yocto Project**는 툴, 메타데이터, 문서 등을 포괄하는 전체 오픈 소스 프로젝트의 이름입니다.
*   **Poky**는 그 프로젝트에서 제공하는 구체적인 **빌드 시스템의 인스턴스**입니다.
*   일반적으로 "Yocto를 설치한다"는 것은 Poky 저장소를 다운로드(`git clone`)하는 것을 의미합니다.

## 5. 기본 워크플로우

Poky를 사용하여 이미지를 빌드하는 과정은 다음과 같습니다.

1.  **다운로드:** `git clone git://git.yoctoproject.org/poky`
2.  **환경 설정:** `source oe-init-build-env`
3.  **빌드:** `bitbake core-image-minimal`

---
*참고: Poky는 데비안(Debian)이나 페도라(Fedora) 같은 바이너리 배포판이 아니라, 소스 코드로부터 시스템을 생성하는 빌드 시스템입니다.*
