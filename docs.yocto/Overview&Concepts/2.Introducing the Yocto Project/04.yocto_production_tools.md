# Yocto Project 프로덕션 도구 (Production Tools)

Yocto Project는 개발 도구 외에도, 장기적인 유지 관리와 빌드 시스템의 핵심 동작을 지원하는 **프로덕션 도구(Production Tools)**를 제공합니다.

## 1. 개요

이 도구들은 주로 배포판 관리자(Distribution Maintainer)나 시스템 통합자(System Integrator)가 레시피를 최신 상태로 유지하거나, 패치를 검증하고, 빌드 시스템의 무결성을 보장하는 데 사용됩니다.

## 2. 주요 도구

### 2.1 Auto Upgrade Helper (AUH)
레시피의 업스트림 버전 업데이트를 자동화하는 도구입니다.
*   **기능:** 새로운 업스트림 릴리스가 있는지 확인하고, 레시피 파일(`.bb`)을 수정하여 버전을 올리고, 새로운 체크섬을 계산하며, 기본적인 빌드 테스트를 수행합니다.
*   **목적:** 수백 개의 레시피를 관리해야 하는 유지 관리자의 부담을 줄여줍니다.

### 2.2 Patchtest
기여된 패치의 품질을 검사하는 도구입니다.
*   **기능:** 패치 파일이 Yocto Project의 메타데이터 스타일 가이드와 커밋 메시지 규칙을 준수하는지 자동으로 테스트합니다.
*   **목적:** 프로젝트에 병합되기 전에 패치의 형식을 검증하여 코드 품질을 유지합니다.

### 2.3 Pseudo
루트 권한을 시뮬레이션하는 도구입니다.
*   **기능:** 호스트 시스템에서 관리자(root) 권한 없이도 파일의 소유권(UID/GID)과 권한을 설정할 수 있는 가상 환경을 제공합니다. `LD_PRELOAD`를 사용하여 시스템 호출을 가로채는 방식으로 동작합니다.
*   **중요성:** BitBake가 타겟 루트 파일시스템을 생성할 때, 호스트의 보안을 침해하지 않으면서 올바른 권한을 가진 파일을 생성하기 위해 필수적으로 사용됩니다.

### 2.4 Error Reporting Tool
빌드 실패 정보를 수집하는 도구입니다.
*   **기능:** `bitbake` 실행 중 발생한 오류를 웹 서버로 전송하여, 다른 개발자들이 동일한 문제를 겪었는지 확인하거나 해결책을 찾을 수 있도록 돕습니다.
*   **설정:** `local.conf` 파일에서 `INHERIT += "report-error"`를 추가하여 활성화할 수 있습니다.

---
*참고: 프로덕션 도구는 직접 호출하여 사용하기보다는 빌드 시스템 내부에서 동작하거나(Pseudo), 자동화된 인프라의 일부로 사용되는(AUH, Patchtest) 경우가 많습니다.*
