# Yocto Project 기본 용어 (Some Basic Terms)

Yocto Project 문서를 읽거나 개발을 진행할 때 자주 접하게 되는 핵심 용어들을 정리했습니다.

## 1. 핵심 구성 요소

*   **BitBake:** Yocto Project의 빌드 엔진이자 작업 스케줄러입니다. 메타데이터를 읽어 의존성을 파악하고 빌드 작업을 실행합니다.
*   **Recipe (`.bb`):** 특정 소프트웨어 패키지를 빌드하는 방법을 정의한 파일입니다. 소스 코드 위치, 패치, 컴파일 옵션, 의존성 등을 기술합니다.
*   **Metadata:** 빌드 시스템이 "무엇을", "어떻게" 빌드할지 정의한 데이터의 집합입니다. 레시피, 설정 파일(`.conf`), 클래스(`.bbclass`), 인클루드 파일(`.inc`) 등을 통칭합니다.
*   **Layer:** 관련된 메타데이터를 논리적으로 그룹화한 디렉토리입니다. 모듈화와 재사용성을 위해 사용되며, 보통 `meta-` 접두사를 가집니다.
*   **OpenEmbedded-Core (OE-Core):** Yocto Project의 가장 핵심적인 메타데이터 레이어입니다. 기본적인 툴체인, 커널, 라이브러리 등의 레시피를 포함합니다.
*   **Poky:** Yocto Project의 레퍼런스 배포판(Reference Distribution)입니다. BitBake, OE-Core, 그리고 기본 레이어들을 통합하여 제공하는 빌드 시스템의 인스턴스입니다.

## 2. 빌드 환경 및 결과물

*   **Build Directory:** 실제 빌드 작업이 수행되는 공간입니다. `source oe-init-build-env` 명령을 실행한 위치이며, 설정 파일(`conf/`)과 빌드 결과물(`tmp/`)이 저장됩니다.
*   **Source Directory:** Poky 저장소를 포함하여 메타데이터와 소스 코드가 위치한 디렉토리입니다.
*   **Image:** 타겟 디바이스에서 실행 가능한 최종 결과물입니다. 부트로더, 커널, 루트 파일시스템 등을 포함합니다.
*   **Package:** 컴파일된 바이너리와 설정 파일들을 배포 및 설치 가능한 형태로 묶은 것입니다. (예: RPM, DEB, IPK)
*   **SDK (Software Development Kit):** 애플리케이션 개발자가 타겟 시스템용 소프트웨어를 개발할 수 있도록 제공되는 툴체인과 라이브러리 모음입니다.

## 3. 작업 및 설정

*   **Task:** BitBake가 레시피를 처리할 때 실행하는 개별 작업 단위입니다. (예: `do_fetch`, `do_compile`, `do_install`)
*   **Configuration (`.conf`):** 빌드 환경, 하드웨어 설정, 배포판 정책 등을 정의하는 파일입니다. (예: `local.conf`, `machine.conf`)
*   **Cross-Compilation:** 호스트 머신(예: x86 PC)과 다른 아키텍처(예: ARM)를 가진 타겟 머신용 코드를 빌드하는 과정입니다.

## 4. 기타 중요 용어

*   **BSP (Board Support Package):** 특정 하드웨어 보드를 동작시키기 위해 필요한 커널, 드라이버, 부트로더 설정을 포함하는 레이어입니다.
*   **Upstream:** Yocto Project 외부의 원본 오픈 소스 프로젝트(예: Linux Kernel, GNU Bash)를 의미합니다.
*   **Sysroot:** 크로스 컴파일 시 필요한 헤더 파일과 라이브러리가 위치하는 논리적인 루트 디렉토리입니다.
