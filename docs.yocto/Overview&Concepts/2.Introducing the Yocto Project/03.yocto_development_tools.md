# Yocto Project 개발 도구 (Development Tools)

Yocto Project는 임베디드 리눅스 시스템 개발을 효율적으로 수행하기 위해 다양한 커맨드 라인 도구와 유틸리티를 제공합니다.

## 1. 개요

이 도구들은 레시피 생성, 소스 코드 수정, 이미지 빌드, 그리고 애플리케이션 개발 환경 구성을 돕습니다. 개발자는 이러한 도구를 사용하여 수동 작업을 줄이고 생산성을 높일 수 있습니다.

## 2. 주요 도구

### 2.1 devtool
`devtool`은 Yocto Project 개발 워크플로우를 단순화하는 강력한 도구입니다.
*   **기능:** 기존 레시피 수정(`modify`), 새 레시피 추가(`add`), 레시피 업그레이드(`upgrade`), IDE 설정 생성(`ide-sdk`) 등을 수행합니다.
*   **장점:** 별도의 레이어를 수동으로 생성하거나 복잡한 환경 설정 없이도, 작업 공간(Workspace)에서 소스 코드를 수정하고 즉시 빌드 및 배포(`deploy-target`)하여 테스트할 수 있습니다.

### 2.2 recipetool
`recipetool`은 소스 코드로부터 자동으로 기본 레시피를 생성하는 도구입니다.
*   **기능:** `recipetool create <source>` 명령을 통해 소스 코드의 빌드 시스템(Autotools, CMake, Python 등)을 분석하고 적절한 `.bb` 파일을 생성합니다.
*   **활용:** 주로 `devtool add` 명령 내부에서 호출되어 사용됩니다.

### 2.3 BitBake
Yocto Project의 핵심 빌드 엔진이자 스케줄러입니다.
*   **기능:** 메타데이터(레시피, 설정 파일)를 파싱하고, 태스크 간의 의존성을 해결하며, 병렬로 빌드 작업을 실행합니다.
*   **사용:** `bitbake <target>` 명령으로 이미지나 패키지를 빌드합니다.

### 2.4 Toaster
BitBake의 웹 기반 인터페이스입니다.
*   **기능:** 빌드 구성 및 실행, 빌드 통계(시간, 디스크 사용량) 확인, 패키지 및 파일 탐색 등을 웹 브라우저에서 수행할 수 있습니다.
*   **장점:** 빌드 프로세스를 시각화하여 초보자가 시스템을 이해하거나 빌드 실패 원인을 분석하는 데 도움을 줍니다.

### 2.5 wic
OpenEmbedded 이미지 생성 도구입니다.
*   **기능:** 파티션 정보를 정의한 `.wks` (Kickstart) 파일을 사용하여 부팅 가능한 디스크 이미지를 생성합니다.
*   **활용:** 복잡한 파티션 레이아웃이나 부트로더 설정이 필요한 경우 `IMAGE_FSTYPES`에 `wic`를 추가하여 사용합니다.

## 3. SDK (Software Development Kit)

Yocto Project는 타겟 하드웨어용 애플리케이션 개발을 위해 두 가지 형태의 SDK를 생성할 수 있습니다.

*   **Standard SDK:** 크로스 컴파일러, 링커, 라이브러리, 헤더 파일 등을 포함하는 독립적인 툴체인 설치 파일입니다.
*   **Extensible SDK (eSDK):** Standard SDK의 모든 기능에 더해, `devtool`을 사용하여 애플리케이션을 Yocto 빌드 시스템과 통합된 환경에서 개발할 수 있도록 지원합니다.
