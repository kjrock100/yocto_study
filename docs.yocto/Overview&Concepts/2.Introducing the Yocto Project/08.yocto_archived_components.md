# Yocto Project 아카이브된 구성 요소 (Archived Components)

Yocto Project는 지속적으로 발전하면서 일부 도구와 구성 요소가 더 나은 대안으로 대체되거나 더 이상 유지 관리되지 않아 **아카이브(Archived)** 상태로 변경되었습니다.

## 1. 개요

이 섹션에 나열된 구성 요소들은 과거에는 사용되었으나, 현재는 더 이상 적극적으로 개발되지 않거나 사용이 권장되지 않는 것들입니다. 레거시 프로젝트를 유지 보수하는 경우가 아니라면, 새로운 프로젝트에서는 최신 도구를 사용하는 것이 좋습니다.

## 2. 주요 아카이브 구성 요소

### 2.1 Hob
**Hob**은 BitBake를 위한 GTK+ 기반의 그래픽 사용자 인터페이스(GUI)였습니다.
*   **기능:** 사용자가 레시피를 선택하고, 패키지를 구성하고, 이미지를 빌드할 수 있도록 도왔습니다.
*   **대체:** 현재는 웹 기반 인터페이스인 **Toaster**로 완전히 대체되었습니다. Toaster는 더 강력한 기능과 원격 빌드 지원을 제공합니다.

### 2.2 Yocto Project Build Appliance
**Build Appliance**는 Yocto Project 빌드 시스템이 사전 설치 및 구성된 가상 머신(VMware, VirtualBox 등에서 실행 가능한 이미지)이었습니다.
*   **목적:** 사용자가 리눅스 호스트 머신을 직접 설정하지 않고도 Yocto Project를 체험해 볼 수 있도록 제공되었습니다.
*   **현재 상태:** 컨테이너 기술(Docker 등)과 `crops/poky` 같은 컨테이너 이미지의 발전으로 인해, 무거운 VM 이미지를 유지 관리하는 것은 효율적이지 않게 되었습니다. 현재는 **CROPS (Cross-Platform Support)** 프로젝트를 통해 Docker 컨테이너를 사용하는 것이 권장됩니다.

### 2.3 EGLIBC
과거 Yocto Project는 **EGLIBC (Embedded GLIBC)**를 기본 C 라이브러리로 사용했습니다.
*   **배경:** EGLIBC는 임베디드 시스템에 최적화된 glibc의 변형이었습니다.
*   **현재 상태:** EGLIBC 프로젝트가 종료되고 glibc로 다시 통합됨에 따라, Yocto Project도 표준 **GNU C Library (glibc)**로 복귀했습니다. 현재는 `musl`과 `glibc`가 주요 C 라이브러리 옵션입니다.
