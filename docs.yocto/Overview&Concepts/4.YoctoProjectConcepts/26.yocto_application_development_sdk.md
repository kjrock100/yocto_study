# Yocto Project 애플리케이션 개발 SDK (Application Development SDK)

Yocto Project는 시스템 이미지뿐만 아니라, 해당 시스템 위에서 동작하는 애플리케이션을 개발하기 위한 **SDK (Software Development Kit)** 생성 기능을 제공합니다.

## 1. 개요

애플리케이션 개발 SDK는 개발자가 Yocto Project 빌드 시스템 전체를 설치하거나 실행하지 않고도, 타겟 디바이스용 애플리케이션을 컴파일하고 링크할 수 있는 독립적인 환경을 제공합니다.

## 2. 주요 구성 요소

SDK는 크게 다음 요소들로 구성됩니다.

### 2.1 크로스 개발 툴체인 (Cross-Development Toolchain)
호스트 머신(예: x86_64)에서 실행되지만, 타겟 머신(예: ARM, MIPS)을 위한 코드를 생성하는 컴파일러, 링커, 디버거 등을 포함합니다.
*   **GCC (GNU Compiler Collection)**
*   **GDB (GNU Debugger)**
*   **Binutils**

### 2.2 라이브러리 및 헤더 (Libraries and Headers)
타겟 이미지(`rootfs`)에 포함된 라이브러리와 헤더 파일들의 복사본(Sysroot)을 포함합니다. 이를 통해 애플리케이션이 타겟 시스템에 존재하는 라이브러리와 정확히 일치하는 버전으로 링크될 수 있습니다.

### 2.3 환경 설정 스크립트 (Environment Setup Script)
SDK를 설치한 후 실행해야 하는 쉘 스크립트입니다.
*   `CC`, `LD`, `CXX` 등의 환경 변수를 SDK 내부의 툴체인 경로로 설정합니다.
*   `PATH`를 업데이트하여 크로스 컴파일러를 바로 사용할 수 있게 합니다.

## 3. SDK의 종류

### 3.1 Standard SDK
*   전통적인 형태의 SDK입니다.
*   툴체인과 라이브러리만 포함하며, 애플리케이션 크로스 컴파일에 최적화되어 있습니다.
*   Makefile이나 Autotools 기반 프로젝트에 적합합니다.

### 3.2 Extensible SDK (eSDK)
*   Standard SDK의 모든 기능을 포함하며, 추가로 `devtool`과 Yocto 빌드 시스템의 일부를 포함합니다.
*   새로운 레시피를 추가하거나, 기존 패키지를 수정하고, SDK 자체를 업데이트하는 기능이 있습니다.
*   `devtool`을 사용하여 워크플로우를 자동화할 수 있습니다.

## 4. SDK 생성 및 사용

### 4.1 생성
BitBake를 사용하여 특정 이미지에 맞는 SDK를 생성할 수 있습니다.

```bash
# Standard SDK 생성
bitbake -c populate_sdk <image-name>

# Extensible SDK 생성
bitbake -c populate_sdk_ext <image-name>
```

생성된 설치 파일(`.sh`)은 `tmp/deploy/sdk/` 디렉토리에 저장됩니다.

### 4.2 설치 및 사용
1.  생성된 쉘 스크립트를 실행하여 호스트 PC에 SDK를 설치합니다.
2.  환경 설정 스크립트를 실행합니다.
    ```bash
    source /opt/poky/3.1/environment-setup-aarch64-poky-linux
    ```
3.  이제 `$CC` 변수 등을 사용하여 애플리케이션을 컴파일할 수 있습니다.
    ```bash
    $CC hello.c -o hello
    ```

## 5. 요약

애플리케이션 개발 SDK는 시스템 개발자와 애플리케이션 개발자의 역할을 분리해 줍니다. 시스템 개발자는 OS 이미지를 빌드하고 SDK를 배포하며, 애플리케이션 개발자는 배포된 SDK를 사용하여 앱 개발에만 집중할 수 있습니다.
