# Yocto Project 구성 요소 (Yocto Project Components)

Yocto Project 빌드 시스템은 여러 핵심 구성 요소들이 유기적으로 결합되어 동작합니다. 이 섹션에서는 BitBake, 레시피, 클래스, 그리고 설정 파일에 대해 설명합니다.

## 1. BitBake
BitBake는 Yocto Project의 핵심 **빌드 엔진**이자 **작업 스케줄러**입니다.
*   **역할:** 메타데이터를 파싱하고, 작업(Task) 간의 의존성을 해결하며, 정의된 순서대로 작업을 실행합니다.
*   **기능:** `make`와 유사하지만, 단일 패키지가 아닌 전체 임베디드 리눅스 배포판을 빌드하도록 설계되었습니다. 파이썬(Python)과 쉘 스크립트가 혼합된 문법을 사용합니다.

## 2. 레시피 (Recipes)
레시피는 개별 소프트웨어 패키지를 빌드하기 위한 지침서입니다.
*   **확장자:** `.bb`
*   **내용:**
    *   소스 코드의 위치 (URL, Git 저장소 등)
    *   패치 적용 방법
    *   빌드 및 런타임 의존성 (DEPENDS, RDEPENDS)
    *   컴파일 및 설치 옵션
*   **구조:** 일반적으로 `recipes-<category>/<package>/<package>_<version>.bb` 형태의 디렉토리 구조를 가집니다.

## 3. 클래스 (Classes)
클래스는 여러 레시피에서 공통적으로 사용되는 빌드 로직을 추상화한 파일입니다.
*   **확장자:** `.bbclass`
*   **역할:** 코드 중복을 방지하고 유지 보수성을 높입니다. 레시피 파일에서 `inherit` 키워드를 사용하여 클래스를 상속받을 수 있습니다.
*   **예시:**
    *   `autotools`: Autotools(configure, make, make install) 기반의 프로젝트를 빌드할 때 사용.
    *   `cmake`: CMake 빌드 시스템을 사용하는 프로젝트용.
    *   `systemd`: systemd 서비스 파일을 설치하고 활성화하는 로직 포함.

## 4. 설정 (Configurations)
설정 파일은 빌드 프로세스 전반에 걸친 전역 변수와 정책을 정의합니다.
*   **확장자:** `.conf`
*   **주요 파일:**
    *   `local.conf`: 사용자가 직접 수정하는 로컬 빌드 설정 (병렬 빌드 수, 디스크 공간 등).
    *   `bblayers.conf`: 빌드에 포함될 레이어들의 경로 정의.
    *   `machine.conf`: 타겟 하드웨어(BSP)에 대한 설정.
    *   `distro.conf`: 배포판 정책(init 시스템, 툴체인 버전 등) 정의.

## 5. 요약
이 네 가지 구성 요소는 다음과 같이 상호작용합니다.
1.  **설정(Configuration)**이 빌드 환경을 정의합니다.
2.  **BitBake**가 설정을 읽고 실행을 준비합니다.
3.  **레시피(Recipes)**가 각 패키지를 어떻게 빌드할지 알려줍니다.
4.  **클래스(Classes)**가 레시피들 사이의 공통 작업을 처리합니다.
